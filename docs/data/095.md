> 场景：一场甲方（周总，要做品牌+经销商铺货）和硬件方案商（朱总/王工一方，"我们这边"）的需求对接会。主题是用乐鑫芯片 + 小钢头/卡通头外壳，基于百度大模型做一台能聊天、能看书识物的 AI 陪伴机器人玩具，谈大模型怎么接、license 怎么算、版权怎么绕、外观怎么改、最后怎么定价铺货。
>

> ⚠️ 这不是一场"大佬分享会"。它是一场<strong>真刀真枪的产品落地谈判</strong>——比任何方法论都值钱的是：你能看到一个 AI 硬件产品从"想法"到"成本表"到"绕版权话术"的全过程，包括它脏的那一面。

## ⏱ 如果只读 10 分钟（7 条带走判断）

1. <strong>AI 陪伴硬件的真正壁垒不是大模型，是"像不像人"的交互体验。</strong> 周总反复对比小智、京东 JAY、悟空机器人：模型谁都能接，但小智"像聊了多年的老朋友"，别家"像和小朋友说话，问一答一、很无趣"。差距全在人设调教、打断逻辑、响应速度这些"软"细节上。
2. <strong>"换大模型"在这套架构里几乎是改几个词的事，但"换厂商"要重新付费。</strong> 方案商点破：百度的两个模型随便切、改 prompt 即可；但接豆包/通义千问这种第三方，要额外签约付费。<strong>license（买百度整套服务）和 token（按量算力费）是两码事</strong>——这是全场最容易被甲方搞混、也最该搞清的一笔账。
3. <strong>加摄像头 OCR，license 成本直接翻倍。</strong> 纯语音版 license 约 40 元/套，加上摄像头识物/识字（OCR），"基本上翻倍"。任何"让它能看"的功能，都要先在成本表上加一行。
4. <strong>"实时视频对话"在端侧基本是假的。</strong> 方案商挑明：所谓视频，后端"最多一秒两张图"，本质是连续拍照。建议第一版只做拍照识图，实时视频对话放二期或砍掉——这是一条能帮你避免给客户画饼的硬约束。
5. <strong>版权是这台机器最大的合规雷区，而这场会给出的"解法"是灰色的。</strong> 想播放全网音乐（周杰伦等）但没版权，于是设计了一套"个人行为"话术：公司只提供 MCP 插件能力，由"兼职员工以个人身份刷机"，再用小红书伪装第三方种草引流——把法律风险转嫁到"用户个人行为"上。⚠️<strong>这是一条结构性合规风险，不是可复用的好方法，照搬有真实法律后果。</strong>
6. <strong>这台机器的真正商业模式是"经销商铺货 + 快速回笼资金 + 反哺二期研发"，不是靠这台硬件赚钱。</strong> 周总的算盘很清楚：硬件成本约 250、含包装约 278，自己留 50%+ 毛利给到经销商约六百多，经销商卖 699–799（对标市面 2000 多）。第一代用来"迅速铺开、回笼资金"，钱赚回来去做基于扣子(Coze)的智能体二期（点外卖/订机票/做 PPT）。
7. <strong>第一版的铁律是"代价最小"：能不动的硬件结构就不动。</strong> 只换头型（小钢头→圆形阿拉伯人造型）、身体弧度微调，内部齿轮用现成标准件不开模，手部电机（空心杯电机十三四块）这种加成本的先砍，5G/4G 做兼容预留。一切为压成本、抢首发。

## 🎯 这篇真正能学什么

这不是学"AI 硬件多火"，而是练<strong>三种眼力</strong>：①<strong>把一个 AI 硬件需求翻译成成本表的能力</strong>——每加一个功能（摄像头/电机/5G），成本表上加多少、license 翻不翻倍，方案商一句句给你算；②<strong>识别"伪需求"和"伪功能"的能力</strong>——实时视频是假的、屏幕触摸找不到场景、手部电机性价比低，懂行的人当场就砍；③<strong>看清一个灰色商业设计是怎么被一步步合理化的</strong>——从"想播全网音乐"到"包装成个人刷机行为"再到"小红书伪装第三方"，整个话术链条值得当反面案例研究。读完别记机器人长什么样，记这三套"翻译/砍需求/识别风险"的思维。

## 📚 深挖：7 个值得单独学的知识点

### 1. license ≠ token：AI 硬件成本表里最该分清的两笔账
甲方周总一开始把 license 和 token 混为一谈，方案商专门停下来解释："license 跟 token 不是一个东西，license 相当于买百度整套服务，用他们家模型不用钱；但用第三方（豆包/通义）要另外付钱。" token 是按量的算力费（聊一年也就几块钱，很低），license 是买"把这套服务包好"的服务费（约 40 元/套）。
<strong>为什么重要</strong>：这是 AI 硬件成本结构的底层框架。混淆这两者会让你完全算错单台成本和切换不同模型的代价。
<strong>你能用</strong>：做任何接大模型的硬件/应用，先把成本拆成"一次性服务/授权费"和"按量算力费"两栏分别算。

### 2. 加一个摄像头（OCR），license 直接翻倍
周总想让机器人"看书识字、识别万物"，方案商立刻反应："加上 OCR 识别，license 基本上翻倍。" 纯语音版约 40，加摄像头识物就到约 80。
<strong>为什么重要</strong>：硬件需求里"让它能看"听起来只是加个摄像头零件，但真正翻倍的是后端识别服务的 license。功能的成本往往不在你看得见的硬件上。
<strong>你能用</strong>：评估"加个 XX 功能"时，永远追问"这功能背后调用的服务/授权费是多少"，而不只算物料 BOM。

### 3. "实时视频对话"在端侧是伪功能——本质是连续拍照
方案商挑明：端侧"新定位性能没那么强"，所谓视频"最多一秒支持两张图"，"所谓的视频其实就是图片"。建议第一版只做拍照识图（没问题），实时视频对话"更麻烦"，效果不好就直接砍。
<strong>为什么重要</strong>：这是懂行的人帮甲方"祛魅"的典型时刻。很多 AI 硬件宣传"实时视频理解"，落地全是低帧拍照。
<strong>你能用</strong>：听到"实时视频/实时理解"先打个问号，问清楚帧率和实现方式，别把伪功能写进自己的产品宣传。

### 4. AI 陪伴的体验核心：打断模式 + 人设 + 响应速度
周总用悟空机器人举的例子最生动："聊好几句后每次都要喊'悟空悟空'才能继续，违背常识，你都不想跟他聊了。" 他要的是两种可语音切换的模式：<strong>关键词打断</strong>（说话时能直接插话打断）和<strong>随时打断/聆听</strong>（多人闲聊场景）。再叠加人设（"设成老师就像老师，设成老朋友就懂你问题背后的原因"）和响应速度（小智因为"卡多、有钱"所以快）。
<strong>为什么重要</strong>：这三件事——交互打断逻辑、人格设定、延迟——才是 AI 陪伴产品"像不像人"的真实变量，模型反而是可替换的。
<strong>你能用</strong>：做对话类产品，把打断/中断逻辑、人设 prompt、首字延迟当成一级体验指标来打磨，别只盯模型榜单。

### 5. 版权绕行的"个人行为"话术链（⚠️作为反面案例学）
想播放无版权全网音乐，这场会设计的"解法"是：①公司官方版本不带这功能；②由"公司的一个兼职员工"以<strong>个人身份</strong>给用户刷机/提供 MCP 插件；③不通过官方客服发，改用小红书让"张三伪装成第三方"种草引流；④客服话术统一为"我们官方不支持，但发现有第三方插件，我们不支持但你用了……那里有个漏洞"。甚至自我类比"早年杀毒软件公司自己放病毒"的商业模式。
<strong>为什么重要</strong>：这是一个把法律风险从公司转嫁到"用户个人行为"的完整设计，逻辑自洽但合规上极其脆弱——"商用就会被告"是他们自己说的。会上一方也明确表态："这个方式可能不太合规""我们不能干法律不允许的事"。
<strong>你能用</strong>：把它当<strong>风险识别训练</strong>——当一个方案需要靠"伪装成个人""话术规避"来成立时，它本身就是核心风险信号。真要做内容/版权，老老实实买授权或用 SD 卡让用户自备，别赌"个人行为"挡箭牌。

### 6. 第一版"代价最小"的硬件减法哲学
为抢首发压成本，这场会做了一连串减法：外观只换头型（小钢头→圆形/阿拉伯人造型，参考瓦力但不做履带）、身体弧度微调让内部结构几乎不动；齿轮用厂家<strong>现有标准件不开模</strong>（开模成本高）；手部上下摆动需要空心杯电机（十三四块/个）→第一版砍掉记到二期；屏幕 1.85 寸不带触摸（"找不到触摸的场景"）。
<strong>为什么重要</strong>：这是硬件创业最实的功夫——每一个"加上去会更好"的功能，都要和开模费、物料成本、首发速度做权衡。能不动的就不动，是控成本的铁律。
<strong>你能用</strong>：做硬件第一版列一张"砍功能清单"，每个功能标"加多少成本/要不要开模/能否推到二期"，默认能砍就砍。

### 7. 经销商铺货的定价飞轮与成本拆解
周总的完整算盘：单台硬件成本约 250（含 4G 插卡 + 5G WiFi 兼容），加 iPhone 档次的包装盒说明书到约 278；自己保 50%+ 毛利，给经销商约六百多；经销商再留自己 100% 毛利空间，终端卖 699–799，对标市面 2000 多。第一代靠经销商网络"迅速铺开、迅速回笼资金"，回款去做基于扣子的智能体二期。
<strong>为什么重要</strong>：这把"硬件怎么定价、毛利怎么在厂商-经销商之间分、为什么对标价要砍半"讲得很透。砍半对标价（699 vs 2000）是给渠道留出营销和让利空间、换铺货速度。
<strong>你能用</strong>：做铺货型硬件，先把成本→出厂价→渠道价→终端价四级倒推清楚，并明确每一级的毛利空间，再决定对标价怎么打。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：需求逐条翻成本表</strong>
- 核心意思：每加一个功能（摄像头/电机/5G），当场问"这要加多少物料成本、license 翻不翻倍、要不要开模"。
- 你可以怎么用：做需求评审时，强制每条功能后跟一个成本数字，没数字不通过。
- 不要误读成：先把功能堆满再砍——应该边谈边算，让成本约束反向塑造需求。

<strong>方法 2：当场砍伪功能</strong>
- 核心意思：实时视频（其实是低帧拍照）、触摸屏（找不到场景）、手部电机（性价比低）这类伪/弱功能，懂行就当场砍到二期。
- 你可以怎么用：第一版列"砍功能清单"，默认能砍则砍，只留撑得起核心体验的功能。
- 不要误读成：所有炫功能都砍——核心差异化（如打断/人设）反而要重投。

<strong>方法 3：兼容设计把决定权留到后面</strong>
- 核心意思：5G/4G 不确定要不要时，硬件先做兼容预留接口，后面按客户群体再插模块。
- 你可以怎么用：对"还没想清要不要"的配置，用兼容/预留代替现在拍板，降低试错成本。
- 不要误读成：什么都做兼容——兼容也有成本（如这里 5G WiFi 加二三十块），要算账。

<strong>方法 4（反面）：识别"靠话术规避"的方案</strong>
- 核心意思：当一个方案要靠"包装成个人行为""伪装第三方种草"才能成立，说明它本身踩了红线。
- 你可以怎么用：把"是否依赖话术规避"当成方案合规体检的一票否决项。
- 不要误读成：这是一套可复用的增长黑客——它是法律风险，会上对方自己都说"商用会被告"。

## 📅 7 天作业

- <strong>Day 1</strong>：拿你正在做的一个产品，列出"模型/技术可替换的部分"和"真正决定体验的部分（如延迟、交互、人设）"，看你押宝押对了没。
- <strong>Day 2</strong>：给产品做一张成本表，把"一次性授权/服务费"和"按量算力费"分两栏拆开算清。
- <strong>Day 3</strong>：列一张"伪功能/弱功能"清单——哪些是营销话术（如"实时")、哪些场景根本用不上，标出可砍项。
- <strong>Day 4</strong>：如果你做硬件，倒推成本→出厂价→渠道价→终端价四级，标出每级毛利，定一个有竞争力的对标价。
- <strong>Day 5</strong>：审视你某个增长/变现方案，问"它是否依赖话术规避或灰色操作才成立"，如果是，重新设计一个合规版本。
- <strong>Day 6</strong>：研究 MCP / 扣子(Coze) 智能体怎么给你的产品接外部能力（点外卖/订票类），设想一个二期增值方向。
- <strong>Day 7</strong>：把你产品的打断/中断/响应延迟体验，跟一个标杆产品（如小智）做一次盲测对比，记录差距。
