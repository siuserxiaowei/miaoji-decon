> 场景：一通约 26 分钟的硬件项目协调电话。甲方（发言人2，"谢总/董事长"）+ 方案商一侧（发言人3 项目/调度，发言人4 硬件工程师 "夏工"，发言人1 商务/小伟）。议题是把已有的"AI促销员" demo 板子快速改造成可用的第一版，顺带捋读书理财（"读书女孩"）项目的合同与交付。
>

> ⚠️ 这场会信息密度中等且偏执行细节——它不是"大佬分享/商业模式"那类知识局，而是一场<strong>真实的甲乙方硬件协作现场</strong>。它的价值不在某个金句，而在：一个甲方怎么在"工期压力 + 成本敏感 + 改动风险"三者之间做权衡，以及和方案商打交道会踩哪些坑。别拿"道法术器势"硬套出宏大叙事，这桌的硬知识是<strong>工程决策与供应链协作</strong>。

## ⏱ 如果只读 10 分钟（6 条带走判断）

1. <strong>第一版的核心决策是"少改快出"</strong>：能不动就不动。屏可能影响现有逻辑，那就保留屏；摄像头、陀螺仪、LED、码表这些不用的元器件，<strong>只在 BOM 上不贴</strong>，不去改 PCB——因为"动 PCB 这边会花很多时间"。
2. <strong>复用现成板子比合并重画更快</strong>：方案商提议直接拿"读书理财/读书女孩"那块板（同样从开源硬件 Sparkle 改出来、元器件基本匹配）改固件，比把 AI促销的三块板合成一块、重新画板<strong>工作量更小、故障更小</strong>，报价是"硬件+软件三个工作日搞定"。
3. <strong>去掉器件 ≠ 省事</strong>：甲方反复确认"如果拿掉元器件会导致固件/软件要动，那我就不拿"。<strong>只在不触发软件逻辑重测时才去掉</strong>，否则宁可留着多花一点物料成本（"我们公司这些加起来十几块钱一片"）。
4. <strong>WiFi 天线是唯一必须动的硬改</strong>：板载天线（PCB 天线）装进铁皮箱会被屏蔽，必须改成外接同轴天线（IPX/IPEX 座 + 穿孔 + 引线）。工程师提示这要换 WiFi 模块版本，"读书女孩"原本就是按外接天线设计的，可对照。
5. <strong>打样找不靠谱的人，代价是整个工期</strong>：甲方两次被"两天打样"的熟人推荐坑（一次给了块光板没贴片、一次拿补偿款了事），结论是<strong>直接找原厂家贴片、能不打样就不打样</strong>，"我这个时间受不了"。
6. <strong>工期已经在超期边缘</strong>：甲方按方案商报的"20 个工作日"排过期，原计划周初定方案，现在还没定就已面临超期，所以坚持"先签个简单独立的合同"把事钉死、再谈分阶段交付。

## 🎯 这篇真正能学什么

练的不是"听大佬讲赛道"，而是<strong>看一个甲方怎么做硬件迭代的工程取舍</strong>。读完该记住三张图：①「改动成本图」——同一个目标（出可用的第一版）有多条路径（重画板 / 合并板 / 复用现成板 / 删器件），每条路径的真实成本不在物料而在<strong>软件重测和打样时间</strong>；②「供应链踩坑图」——打样/贴片/物料这条链上，熟人推荐和"两天速成"承诺是高发坑；③「先签合同再深聊图」——工期紧张时，先用一纸简单合同把责任和时间钉死，反而是保护工期。

## 📚 深挖：6 个值得单独学的知识点

### 1. "少改快出"——MVP 阶段的改动哲学
甲方的主轴一句话："这次做最简单的改动。"具体表现为：屏可能影响现有逻辑 → 干脆保留屏；不必要的元器件 → 去掉；唯一必须调的只有 WiFi 天线。背后的判断是：<strong>第一版只要"够用"</strong>（能拍照、能兑换、能发交通费），优化是第二阶段的事。
- <strong>为什么重要</strong>：很多人做硬件/产品第一版就想做全、做美，结果工期失控。这场会展示了一个克制的甲方怎么把"想要的迭代东西全放到一边"。
- <strong>你能用</strong>：列出你这一版"必须有"和"可以等"的功能，凡进不了"必须有"的，连带它的器件/逻辑一起延后。

### 2. 删器件的真成本是"软件要不要重测"，不是物料钱
全场出现至少三次同一逻辑：要不要拿掉摄像头/屏/LED/陀螺仪/码表？工程师（夏工）的判断标准很清楚——<strong>纯在 BOM 上"不贴"、不动 PCB 的，随便去；一旦"不接屏、不接摄像头"会让"软件逻辑要重新试过一下"，那就要算进三个工作日的调整里</strong>。甲方因此反复说"如果会导致固件或软件要动，那我就不拿"。
- <strong>为什么重要</strong>：这是硬件协作里最容易误判的成本项——大家盯着 BOM 上省的几块钱，忽略了它触发的固件/软件返工和重测，后者贵得多。
- <strong>你能用</strong>：任何"砍配置省成本"的提议，先问对方一句"这会不会要改固件/软件、要不要重测"，再决定砍不砍。

### 3. 复用同源板 vs 合并重画——为什么"借现成板"反而风险更小
发言人3 提的方案：直接拿"读书理财/读书女孩"那块板来改，因为它和 AI促销员一样是从 <strong>Sparkle 开源硬件</strong>改出来的，元器件基本匹配（差异主要是音量按键、按键拍照 vs 定时拍照）。对比另一条路——把 AI促销现有的三块板合成一块、重新备料、重新画板——后者"工作量反而会大"。双方判断复用同源板"故障更小"，报价"硬件调度软件全部加起来三天搞定"。
- <strong>为什么重要</strong>：同源/同架构的存量资产是被低估的加速器。"重新画板"听起来是更彻底的方案，但它带来重新备料、重新打样、重新调试的全链路成本。
- <strong>你能用</strong>：盘一下你手上有没有"同底座"的旧项目，能借壳改固件就别从零开板。

### 4. WiFi 天线：板载 vs 外接同轴，被铁皮箱逼出来的硬改
这是全场唯一确定要动硬件的点。原因很物理：成品贴在一个<strong>铁皮箱子外面</strong>，而 WiFi 热点在箱子里面，板载（PCB）天线会被金属屏蔽，所以必须引出一个<strong>同轴天线座（IPX/IPEX）</strong>，穿孔、接外接天线、用引线连起来。工程师补充了两个关键细节：①模块上本就留了 IPX/IPEX 同轴接口，找对应的同轴天线即可；②但要看实际贴的是哪个模块版本——如果厂家贴成了"带板载天线"的版本，就要<strong>换模块</strong>（BOM 上换型号）。会末发言人4 还发现"读书女孩"那批可能被贴成了板载天线，要让对方改回外接天线版本。
- <strong>为什么重要</strong>：这是个典型的"机构/材料约束反向决定电路设计"的例子——金属外壳、模块版本、天线类型环环相扣，一个没对上就连不上网。
- <strong>你能用</strong>：金属外壳/封闭结构内的无线设备，提前确认天线方案（外接同轴 / FPC 软天线）和模块版本，别等贴片完才发现连不上。

### 5. 打样/贴片供应链的踩坑实录
甲方讲了两段血泪："找的人告诉我两天能打出来，结果搞了半天给我一个光板（没贴片），还得重新找人贴片"；另一段"4 到 5 天嫌慢，找了个说两天的，最后拿补偿款了事，把我气得鼻子都歪了"。他还点破一个机制：<strong>推荐"两天速成"的熟人，往往自己就是做这个采购/中介的</strong>，所以才敢拍胸脯。结论是：<strong>能找原厂直接贴片就别打样</strong>（"找他直接来贴片，我就懒得打样"）。
- <strong>为什么重要</strong>：硬件项目的隐形杀手不是设计而是供应链交付节奏，"熟人推荐 + 速成承诺"是高发坑。
- <strong>你能用</strong>：建立贴片/打样供应商白名单，对"两天速成"保持警惕；关键件优先走验证过的原厂渠道。

### 6. 工期管理：先签简单独立合同，再谈分阶段交付
甲方按方案商报的"20 个工作日"排过时间，原计划周初定方案，"现在还没定下来，其实我这边已经面临超期了"。他的应对不是继续空谈，而是：①要求"最好独立的一个项目，简单的一个合同"，把这次迭代单独钉死；②"先把合同定了，我再约大家聊怎么分阶段交付，让整体工期不受影响"。同时商务侧承诺"明天十点左右"给合同样本，甲方要 <strong>Word 版本</strong>，按内部规范改完用<strong>红字标出</strong>回给对方确认。
- <strong>为什么重要</strong>：工期紧张时，"先签合同把责任时间钉死"不是官僚动作，而是保护工期——它把口头共识变成可追责的节点。
- <strong>你能用</strong>：多方协作且工期紧时，先用一纸轻量合同锁定范围/时间/付款，再在合同框架内谈分阶段交付。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：用"是否触发软件重测"给改动排序</strong>
- 核心意思：硬件改动的真成本在下游返工（固件/软件重测、重画板、重打样），不在 BOM 物料。
- 你可以怎么用：每个改动提案先标注"会不会动固件/软件"，会动的进高成本档，不动的（如只是 BOM 不贴）随便做。
- 不要误读成：物料越省越好——为省几块钱触发一轮软件重测，得不偿失。

<strong>方法 2：借同源板改固件，而不是从零画板</strong>
- 核心意思：同底座（同款开源硬件改出）的旧项目板，改引脚+少量逻辑就能复用，3 天可成。
- 你可以怎么用：开新项目前先翻存量项目，有同架构就借壳改固件。
- 不要误读成：所有旧板都能直接拿——必须是同源、元器件基本匹配的才省事，差异大反而更乱。

<strong>方法 3：能原厂贴片就别打样</strong>
- 核心意思：第三方打样的"速成承诺"不可靠，原厂直接贴片省掉一轮交付风险。
- 你可以怎么用：关键件走验证过的原厂渠道；不得不打样时，把"光板/未贴片"风险写进沟通。
- 不要误读成：所有打样都不要——验证新设计时打样仍必要，这里说的是已成熟的板别再绕第三方。

- <strong>Day 1</strong>：给你手上的项目列一张"必须有 / 可以等"功能清单，把"可以等"的功能连同器件一起标注延后。
- <strong>Day 2</strong>：对每个待砍器件，问工程师一句"这会不会动固件/软件、要不要重测"，按答案重排砍/留。
