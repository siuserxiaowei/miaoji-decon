2. <strong>方案商反复强调：功能不重要，生态位、抢占速度、IP 和营销才重要。</strong> 这是乙方（一个做了很多款产品的人）的真实判断，也是这通电话最值钱的一句话。
3. <strong>"定制"和"快"是矛盾的。</strong> 想要跟别人不一样的功能（定制），就别指望 40 天；想快，就先吃标准化基础版。手头单子多还要再排队。
4. <strong>甲方没出结构/外壳，是项目卡点。</strong> 方案商默认"客户出结构和设计，我们才照着做"；甲方提出可以自己用 3D 打印先凑合，但"板子不能太大、得塞得进玩具"是硬约束。
5. <strong>交付物清单要问清楚：服务器 + 服务器代码 + 小程序 + 硬件 + 硬件软件，外加各自研发费。</strong> 别天真以为"租个阿里云调个 API 就能用"——服务器代码是要单独写、单独收费的。

## 📚 深挖：6 个值得单独学的知识点

### 1. 一款 AI 语音陪伴硬件的"基础版物料栈"
方案商把基础版讲得很具体，可以直接当清单背：
- <strong>主控</strong>：ESP32-S3（ASR 听成"ESP3 S3"/"ESB32S3"，应为乐鑫 ESP32-S3，自带 WiFi）。
- <strong>音频</strong>：ES8311 音频编解码芯片（ASR 听成"ESES8311"），负责音频输入输出。
- <strong>交互按键</strong>：4 个——开关、音量加、音量减（讲到 4 个但只点了 3 个名）。
- <strong>联网</strong>：WiFi 配网；4G 也能做（4G 版相当于另一块板子，因为 ESP32-S3 本身只带 WiFi）。
- <strong>大脑</strong>：连服务器、连大模型做对话。
<strong>为什么重要</strong>：这套"主控 + 音频 codec + 按键 + 联网 + 大模型对话"几乎是当下所有 AI 语音玩具/陪伴硬件的最小公约数，方案商说这块"做得很熟，40 天能交付"。<strong>你能用</strong>：评估任何 AI 语音硬件报价时，先核对它是不是就这套标准件——是的话别为"功能"溢价。

### 2. 基础版 vs Pro 版怎么切（这是全场最实的方法）
现场达成的切法很清楚：
- <strong>进基础版</strong>：标准件 + 软件能直接调的功能。双屏表情显示（两个显示器接口显示两只眼睛/表情）讨论后倾向放基础版；LED 灯"可以往上加，也能进基础版"。
- <strong>进 Pro 版</strong>：需要<strong>额外硬件传感器</strong>或<strong>大模型深度定制</strong>的功能。典型就是"抚摸宠物"——摸一下头它动两下，要贴触摸传感器，还可能要大模型配合做反应差异（痒痒了反应不同），所以"算到 Pro 里"。
判断逻辑：<strong>不加新硬件、软件能搞定的 → 基础版；要加传感器或要定制大模型的 → Pro 版。</strong>
<strong>为什么重要</strong>：这是一条可迁移的产品拆分原则——按"是否引入新的硬件依赖/定制成本"切版本，而不是拍脑袋。<strong>你能用</strong>：你做任何硬件/功能排期，先问"这个功能要不要动硬件 BOM 或要不要定制模型"，要就往后放一版。

### 3. "定制"和"快"是一对硬矛盾
方案商讲得很直白："你又是定制功能的，又可以给你很快做出来"是不可能的；手头单子不多时基础版 40 天能出，单子多就得两个月。甲方想要"稍微和别人不一样"，方案商的回应是"对话这块基本都类似，只是基本功能"，言下之意：想差异化得另花时间。
<strong>为什么重要</strong>：这是外包/产品里永恒的三角（快、好/定制、便宜不可兼得）的具体一例。<strong>你能用</strong>：提需求前先自己排序——这次到底是要"快上市抢位"还是"做出差异点"，别两个都要然后怪乙方慢。

### 4. 结构/外壳是被低估的卡点
方案商默认工作流是"<strong>客户先做好结构和设计，我们再照着做电路板</strong>"。甲方结构还没出，方案商当场点出风险："你们玩具不知道多大，板子可能塞不进去。" 甲方的解法：先不管外壳，方案商出最基础的板子，甲方自己用 <strong>3D 打印</strong>简单分一下塞进去——但前提是"<strong>板子不能太大</strong>"。
<strong>为什么重要</strong>：硬件不是只有"功能"，物理尺寸/结构装配是真实约束，软件人最容易忽略。<strong>你能用</strong>：做硬件先把"目标外壳/尺寸"丢给方案商，至少给个板子尺寸上限，别等做完发现塞不进去。

### 5. 报价里到底包含什么（别以为租个云就完事）
方案商把交付物和收费项拆得很细，值得逐条记：
- <strong>交付物</strong>：服务器 + 服务器代码 + 小程序 + 硬件 + 硬件上的软件。
- <strong>收费项</strong>：服务器软件研发费、小程序研发费、硬件研发费。
- <strong>小程序</strong>：一般"送"一个，但只含<strong>配网</strong>这一个功能；要加别的功能，时间和钱都加。
- <strong>服务器</strong>：需要客户<strong>自己租赁</strong>（如阿里云），且<strong>量产时</strong>的云成本另算（本次还没谈批量）。
- 一个反面教材（方案商讲的真实案例）："昨天有个客户以为直接租个阿里云、调个 API 就能用，不用写服务器代码"——错，<strong>服务器代码是要单独写的工作量</strong>。
- 大模型报价（前一天报给魏总的预算）<strong>不含定制</strong>；定制知识库大模型另算。
<strong>为什么重要</strong>：这是一份 AI 硬件外包的"隐藏费用地图"。很多人只看到"一个硬件多少钱"，漏掉服务器代码、小程序扩展、定制模型、量产云成本。<strong>你能用</strong>：拿到任何 AI 硬件报价，逐项追问这 6 块是否包含、谁来租云、量产成本谁担。

### 6. 深度定制大模型：贵、慢、且要看 ROI
甲方问"能不能做深度定制大模型（带知识库、私有化）"。方案商用一个真实案例兜底：
- 给一家做海外医疗手环（佩戴在病人身上采体征数据）的 G 端客户，做了<strong>私有化大模型 + 知识库 + 本地化部署</strong>；
- 价格"将近 <strong>40–50 万</strong>"，工期"将近 <strong>8 个月</strong>"（对方流程慢）；
- 前提是<strong>客户得提供数据</strong>，方案商负责清洗——"没有数据怎么做知识库、怎么让大模型回答？"
- 方案商对这个宠物项目的判断：<strong>数据难批下来、又贵又慢</strong>；而"宠物只卖几百块，ROI 太低，划不来"，这事要等运营到一定规模再说。深度定制要好效果得"做 agent 那种"，价格很高。
<strong>为什么重要</strong>：这给"要不要上私有化/定制大模型"提供了一个量级感（几十万、几个月、依赖甲方数据）和一条决策线（<strong>先看客单价和规模撑不撑得住</strong>）。<strong>你能用</strong>：别一上来就喊"我要定制大模型"；先用通用大模型跑基础版验证市场，规模起来、数据到位再谈深度定制。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：按"是否动硬件/定制"切基础版与 Pro 版</strong>
- 核心意思：软件能搞定的进基础版，要加传感器或定制大模型的进 Pro 版，先交付能跑的基础版。
- 你可以怎么用：把功能清单过一遍，逐条标"基础/Pro"，先发基础版抢时间。
- 不要误读成：Pro 版功能不重要——只是它们成本/工期重，要分期，不是砍掉。

<strong>方法 2：给外包预留"调试回合"的时间 buffer</strong>
- 核心意思：甲方把 40 天极限往一个月谈，是为了留出"到我这调试、遇到问题再问你"的反复时间。
- 你可以怎么用：排期时别按乙方的"最快能做完"签，留一段双方联调时间。
- 不要误读成：工期越长越好——是留必要 buffer，不是放任拖延。

<strong>方法 3：把报价拆成模块逐项确认边界</strong>
- 核心意思：服务器/服务器代码/小程序/硬件/硬件软件/定制模型各自计费，云租赁和量产成本要单列。
- 你可以怎么用：拿到报价后做一张"含/不含"表，逐项打勾，特别盯"服务器代码""小程序扩展功能""量产云成本"。
- 不要误读成："总价一口价"就安全——隐藏项往往在量产和定制阶段才爆出来。
