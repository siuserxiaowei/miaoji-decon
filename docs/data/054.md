> 场景：一通约 16 分钟的两方电话。需求方（一家想做"AI 养成电子宠物"的小团队，硬件靠一个读电子专业的兼职工程师）向一家广东顺德的手办/3D 模型定制厂提需求，谈"小智 AI + Mongo Go"缝合体的硬件结构、外壳、表情和换身体玩法。本质是一场<strong>硬件产品定义 + 供应链对接</strong>的早期 kickoff，不是大佬分享，信息密度中等。
>
>
> ⚠️ 这通话很容易被当成"客户咨询定制手办"流水账归档——但里面藏着一张完整的<strong>低壁垒 AI 硬件怎么定义产品、怎么排坑</strong>的小白教学图。

## ⏱ 如果只读 10 分钟（6 条带走判断）

1. <strong>产品定义就是一句"缝合体"</strong>：mongo go（一款桌面养成机器人）+ 小智 AI = 会对话、有表情、能蓝牙控车机的车载/挂饰宠物。把竞品当坐标系，用"A+B"一句话锁死 MVP，是小团队最快的产品定义法。
2. <strong>没有技术壁垒 = 拼速度，不是拼技术</strong>。需求方自己反复说"这个产品不存在太大技术壁垒""国内做的人还不多""不够快后面就难搞了"。低壁垒赛道的胜负手是抢时间窗，不是攒技术。
3. <strong>后台直接买小智官方企业接口，别自研</strong>。"购买他们的服务会简单很多"——能买的环节就买，把自研集中在真正差异化的地方（这里是表情和外壳形态）。
4. <strong>差异化点是"表情 + 换身体"，不是 AI</strong>。小智默认表情"拿不出手"，对工业成品/手办是硬伤；真正的卖点是多套精致表情 + 头身分离、换身体切换角色（换猫身体→自动切猫表情/猫叫语音）。AI 是底座，形象/情绪才是用户买单的东西。
5. <strong>结构约束先行，越小越好</strong>。头部直径目标压到 5cm 以内、能到 4cm 最好；屏幕从对标的 1.83 寸放大到 2.x 寸；电池和无线充电塞身体、其余模块（屏/板/蓝牙/4G）塞头部。市场调研结论是"挂饰类整体往小做"——尺寸是被市场而非技术决定的。
6. <strong>小智生态最大的痛点是"没整合"</strong>：蓝牙、4G、红外各路大神各做各的开发板，没人合在一起。谁先把零散能力整合成一个能卖的成品，谁就有窗口。

## 🎯 这篇真正能学什么

这场会练的是<strong>"如何给一个低壁垒 AI 硬件做产品定义"的眼力</strong>。读完该记住三张图：①用"竞品 A + 竞品 B 缝合体 + 一句差异化"快速锁 MVP；②在没有壁垒的赛道里，速度、可买即买、聚焦差异化，比堆技术重要；③早期 kickoff 该把哪些参数（尺寸/屏/电池/接口/留扩展位）和哪些风险（死机、保密、版权暗线）提前钉死。别记"他们要做个宠物机器人"，记这套定义和排坑的动作。

## 📚 深挖：7 个值得单独学的知识点

### 1. 用"缝合体"定义 MVP：mongo go + 小智 AI
全场的产品定义就一句话——"mongo go 跟小智 AI 的一个结合体"。mongo go（ASR 可能把 Mango/某桌面养成机器人识别成此名，<strong>名称待核</strong>）提供"养成桌面宠物"的形态和交互范式，小智 AI 提供对话能力。
- <strong>为什么重要</strong>：小团队最怕产品定义发散。用两个已存在的竞品当锚点，"取 A 的形态 + B 的能力 + 我加一点差异化"，能在一句话内把 MVP 边界画死，工程师和供应链都立刻知道要做什么。
- <strong>你能用</strong>：给你的产品也写一句"X + Y 缝合体 + 我的一点不同"，写不出来说明定义还没收敛。

### 2. "没有技术壁垒"是判断，不是抱怨——它直接决定打法
需求方至少三次强调没壁垒："这个产品不存在太大的技术壁垒""国内做的人还不多""如果我们不够快，到后面这个产品就很难搞了"。
- <strong>为什么重要</strong>：壁垒高低决定资源往哪压。没壁垒 = 谁都能做 = 拼的是<strong>速度、供应链、形象差异化、品牌</strong>，而不是技术深度。把工程精力押在"自研一个谁都能调的 AI"上是错配。
- <strong>你能用</strong>：进任何赛道先诚实判断壁垒。判断"低壁垒"后，立刻把资源从"攒技术"转到"抢时间 + 做别人抄不走的形象/品牌"。

### 3. 能买的环节就买：小智官方企业接口
"我们可以跟小智官方谈，让他开企业接口，直接购买他们的服务会简单很多。"
- <strong>为什么重要</strong>：这是"自研 vs 采购"的经典决策。后台/对话能力不是你的差异化，自研只会拖慢速度、增加维护。把它当成可采购的基础设施买进来，省下的时间投到真正差异化（表情、外壳、换身体）。
- <strong>你能用</strong>：列一张"该买 vs 该自研"清单——凡是不构成差异化、市场已有成熟供给的，优先买。

### 4. 真正的差异化：多表情 + 头身分离换角色
现场对小智的吐槽很关键："小智还停留在很原始的阶段，对话出文字、屏幕角落一个小小的默认表情，对手办/工业成品来说绝对拿不出手。"于是差异化定在两点：①做多套精致表情（猫/狗/熊猫等）；②头身分离——硬件集中在头部，身体只放电池+无线充电，<strong>通过换身体切换角色</strong>（换上猫身体，语音/表情自动切成猫模式）。头身用一颗"很简单的射频芯片"通信识别。
- <strong>为什么重要</strong>：在低壁垒赛道，AI 对话能力人人都有，<strong>形象和情绪体验</strong>才是抄不走且能让用户买单的。而"换身体=换角色"把外观差异化顺手变成了<strong>可重复销售的配件 SKU</strong>（喜欢猫买猫身体，喜欢狗买狗身体）——一条潜在的后端收入线。
- <strong>你能用</strong>：找你产品里"功能之外、情绪/形象之上"的差异点；想想能不能把它做成可换、可复购的配件，而不是一次性卖死。

### 5. 硬件项目的正确顺序：先冻结结构尺寸，再设计板子
反复出现的工作流："板子要设计的话要匹配模型……我们会把预计尺寸给到你们，然后内部做镂空。"头部定 7.5–8cm 圆形（后又压到目标 5cm 内、力争 4cm），屏幕从对标的 1.83 寸放大到 2.x 寸，电池+无线充电放身体。
- <strong>为什么重要</strong>：硬件不是软件，<strong>外壳和结构是硬约束</strong>。先定外形尺寸 → 反推电路板和电池能塞多大 → 做镂空，顺序错了就返工。而尺寸的决策依据是"客户和市场调研发现整体往偏小做"——是市场而非工程师拍板。
- <strong>你能用</strong>：做任何带外壳的硬件，先冻结结构尺寸和关键件（屏、电池）尺寸，再让电路去适配；尺寸优先听市场。

### 6. 早期就要钉死的"排坑项"：电源键 + 扩展接口
两个细节很专业：①"要预留一个开关接口，防止死机……要能完全断电、重启，因为小智用起来偶尔会死机"——上一个物理电源/重启键。②表情和功能都"留一个接口给我们后续扩充修改"。
- <strong>为什么重要</strong>：这是把"用户实际会遇到的故障（死机）"和"产品后续迭代"在 kickoff 阶段就设计进硬件。物理断电键是软件死机的兜底；扩展接口是给未来留活路，避免一版定死。
- <strong>你能用</strong>：硬件 spec 里永远留两样——故障兜底（强制断电/复位）和扩展接口；别等出问题、要迭代时才发现没留位置。

### 7. 小智生态的真实状态：能力齐全但"没整合"
"这个方案小智其实有，但他没完善、没整合在一起。几十个开发板，有人做蓝牙、有人做 4G、有人做红外，各做各的，没人整合。"
- <strong>为什么重要</strong>：这是开源/社区生态的典型机会结构——<strong>零件都在，但没人攒成能卖的成品</strong>。整合本身就是价值：把散落的蓝牙/4G/红外能力 + 精致外壳 + 表情，打包成一个普通用户能直接买的产品。
- <strong>你能用</strong>：盯任何活跃的开源/开发板生态，找"能力已具备但没人做成消费级成品"的缝隙——整合者吃这块红利。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：用"竞品缝合体"一句话定义 MVP</strong>
- 核心意思：取两个已存在竞品的形态/能力 + 一句自己的差异化，锁死产品边界。
- 你可以怎么用：写"X + Y + 我的不同"，写不出就继续收敛。
- 不要误读成：照抄竞品——缝合体的灵魂是那"一句差异化"，没有它就是抄。

<strong>方法 2：低壁垒赛道用"该买 vs 该自研"清单分配资源</strong>
- 核心意思：非差异化、市场已有成熟供给的，全部买；自研只押差异化。
- 你可以怎么用：把模块逐个打勾"买/自研"，自研项不超过 1–2 个核心差异化。
- 不要误读成："买"是偷懒——在拼速度的赛道，乱自研才是真风险。

<strong>方法 3：硬件先冻结结构尺寸，再排电路</strong>
- 核心意思：外形/关键件尺寸是硬约束，先定它，电路去适配镂空。
- 你可以怎么用：kickoff 就给出预计尺寸+电池尺寸，让结构和电路并行不返工。
- 不要误读成：尺寸由工程师定——这里明确是市场调研（往小做）定的。

## 📅 7 天作业

- <strong>Day 1</strong>：把你手上的产品写成一句"X + Y 缝合体 + 一句差异化"，逼自己收敛 MVP。
- <strong>Day 2</strong>：诚实判断你赛道的壁垒高低；若低，写下"为什么我能比别人快"的 3 条理由。
- <strong>Day 3</strong>：列"该买 vs 该自研"清单，圈出最多 2 个真正构成差异化、值得自研的模块。
- <strong>Day 4</strong>：找出你产品"功能之外"的差异点（形象/情绪/玩法），想一个能做成可复购配件的版本。
- <strong>Day 5</strong>：如果是硬件，画出结构尺寸约束（外形/屏/电池），标出"先冻结"的几个数。
- <strong>Day 6</strong>：给产品补两条排坑 spec——故障兜底（强制断电/复位）+ 一个扩展接口。
- <strong>Day 7</strong>：扒 2 个对标竞品的众筹页/视频，记下它们的形态、尺寸、定价和你能差异化的点。
