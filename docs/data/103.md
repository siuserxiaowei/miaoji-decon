## 📚 深挖：5 个值得单独学的知识点

### 1. SDK vs API：方案商最核心的一句价值锚
原话："别人家给的都是 API，你要做 API 就离不开了（要自己做大量开发），开发周期很长；我们是直接做 SDK 的，你开发起来就特别快。"
- <strong>是什么</strong>：API 是只给你接口、底层活儿你自己干；SDK 是把一整套能力（声学识别、配网、智能体切换、后台对接）打包好，你"嵌进去"就能用。方案商把"集成度更高"翻译成客户最在意的"开发更快"。
- <strong>为什么重要</strong>：这是所有"卖工具链/卖中间层"生意的通用话术结构——<strong>不要卖你做了什么，要卖对方因此省了什么</strong>（时间/人力/试错）。同时高集成度=高切换成本，客户用上后就不容易换走。
- <strong>你能用</strong>：你卖任何 B 端方案，把"我们的技术特性"逐条翻译成"你因此节省的开发周期/人力成本"。

### 2. 数据归谁：硬件方案合作里比价格更要命的条款
客户全程反复问"用户信息在你那边吗""我能不能拿到用户数据"。方案商给了两套：
- 方案一：直接用方案商的后台（青云牙/CAI 后台，ASR 不准），客户无感、零投入，但<strong>用户名/手机号等数据都在方案商那边</strong>。
- 方案二：客户自建一套"小后台/独立的一套"，<strong>后台对后台对数据</strong>，数据归客户，但需要研发投入、客户得有自己的 APP/小程序。
- <strong>为什么重要</strong>：硬件只是一锤子买卖，<strong>数据和用户才是能持续变现（续费/增值/广告）的资产</strong>。谁掌握后台谁掌握未来收入。方案商默认想把数据留在自己这。
- <strong>你能用</strong>：评估任何"帮你做整套方案"的合作时，第一条就问：做完之后用户数据/账号体系归谁、能不能私有化部署。

### 3. "无订单不驱动研发"：定制功能的硬规矩
关于"绕开 APP、设备直接弹二维码配网"，方案商说："之前有客户提过要，但他没订单，没办法驱动研发。" 但又补一句"你这个单可以这么改"。
- <strong>是什么</strong>：方案商对非标定制的态度——光提需求不下单，不做；有实单了，才排研发。
- <strong>为什么重要</strong>：这是供应链/方案商行业的普遍规则。买方若想要定制功能，<strong>得用订单量去换</strong>，否则永远排不上。也提醒买方：别把"对方说能做"当成"对方会免费做"。
- <strong>你能用</strong>：找代工/方案商要定制功能时，要么用订单量谈，要么准备为研发单独付费，别期待白嫖。

### 4. 选品判断：用销售数据否掉"看起来火"的品类
方案商原话（大意）："毛绒玩具是很棒，但你去看一下各家的销售数据并不怎么样，同质化太严重，都是一样的东西。" 他们的结论是押"开学季学习机 + AI 闹钟"。
- <strong>是什么</strong>：不被赛道热度迷惑，而是回到"各家真实销售数据"做选品。
- <strong>为什么重要</strong>：AI 硬件最容易被"风口叙事"带跑（卢卡博士火 → 一堆人做摄像头机/盒子）。真正该看的是同品类<strong>实际卖动了没有、是否已经红海同质化</strong>。
- <strong>你能用</strong>：选品/做内容前，先找同行真实销量/榜单数据，验证"它真的卖得动"，而不是"它看起来很火"。

### 5. iOS 成本优势：一个会过期的"窗口型"卖点
方案商把"存储涨价、主控涨价、安卓整机涨价，所以我们做 iOS（系统）便宜"当成本优势抛出。
- <strong>是什么</strong>：在元器件涨价周期里，特定技术路线临时变得相对划算。
- <strong>为什么重要</strong>：这类卖点<strong>依赖行情，不是长期规律</strong>——涨价潮一过优势就消失。听任何"成本优势"都要分清是结构性的还是周期性的。
- <strong>你能用</strong>：评估供应商"成本优势"时，追问一句：这是它的长期结构优势，还是只是当下行情的临时窗口？（此处 ASR 把系统名识别得很乱，"iPhone/iOS 系统便宜"在带屏 AI 硬件语境下存疑，需核实其真实指代。）

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：把技术特性翻译成对方的收益</strong>
- 核心意思：客户不关心你给 SDK 还是 API，只关心"我能多快做出来、省多少人"。
- 你可以怎么用：列出你方案的每个技术点，旁边写一句"对方因此节省了什么"。
- 不要误读成：堆技术参数显得专业——参数越多客户越无感。

<strong>方法 2：合作前先锁死"数据/版权/定制收费"三条款</strong>
- 核心意思：硬件方案合作真正的雷区是数据归属、版权归属、定制是否额外收费。
- 你可以怎么用：谈判清单第一页就放这三条，白纸黑字写死。
- 不要误读成：先把产品做出来再谈条款——做完才发现数据/版权不在自己手上就晚了。

<strong>方法 3：选品用真实销售数据，不用赛道热度</strong>
- 核心意思：火的赛道（如毛绒玩具）真实销量可能很差且已同质化。
- 你可以怎么用：选品前先扒同行榜单/销量，确认"真的卖动"。
- 不要误读成：哪个赛道讨论度高就进哪个——讨论度≠成交量。
