## ⏱ 如果只读 10 分钟（5 条带走判断）

1. <strong>喇叭声音小，九成不是喇叭不够大，是没留出音口</strong>。本场最大的争论：现场喇叭被直接打螺丝压在板子底下、没留出音口，声音从缝里漏出来，"办公室听得见，挂车里根本听不见"。结论是先解决出音口和朝向，而不是换更大的喇叭。
2. <strong>腔体比尺寸更决定音质，小喇叭未必差</strong>。设计方反复劝：不一定非用大喇叭，小喇叭只要腔体设计得当、共振传得出来，人声依然清晰；这台玩具主要播报/讲话、极少放音乐，小喇叭够用。
3. <strong>电池按真实续航倒推，别凭感觉塞大的</strong>。实测 1200mAh 配选定喇叭可连续约 3 小时；而设备定位车载、基本插电使用，1 小时续航就够——所以电池被判定"偏大"，体积可砍一半，给结构腾空间。
4. <strong>屏幕会严重衰减雷达信号，雷达不能藏在屏幕后面</strong>。一个关键纠错：雷达原本想放头部，但头部有屏幕，玻璃会让雷达信号大幅衰减，于是确认雷达放身体、且身体那块只需一小块板。
5. <strong>先建"外观模型"塞进确定的元器件，再做结构，能避免从零重做</strong>。设计方的核心工作法：拿已确定的屏幕、电池、喇叭尺寸先在外观模型里占位，定型后再进结构阶段对接，"否则整个模型从零做非常麻烦"。

## 📚 深挖：6 个值得单独学的知识点

### 1. 喇叭"声音小"的真正病因：出音口，不是功率
现场最反复的一个点：喇叭被直接用螺丝固定在电路板底部、压在下面，<strong>没有预留出音口</strong>，声音只能从缝隙漏出，结果"在办公室听得见，挂到车里就听不见"。设计方一再纠偏——不是喇叭太小，是声学结构没做。
<strong>为什么重要</strong>：这是硬件新手最常见的归因错误，一听声音小就想换更大喇叭/加功率，其实先要看出音口有没有、朝向对不对、腔体封没封死。本场还给了朝向经验：喇叭放身体两侧、出音口朝两侧，声音会比朝下/朝后大。
<strong>你能用</strong>：听感不达标时，先排查"出音口 + 朝向 + 腔体密封"，再考虑换料。

### 2. 腔体共振决定音质，小喇叭+好腔体优于大喇叭+烂结构
设计方解释：大喇叭低音更足、音质更好，但塞进去要留窗口和支撑结构、占空间；而小喇叭只要"腔体比较大、共振能传出来"，人声依旧清晰。这台玩具主用于播报/讲话、"播放音乐的场景比较少"，所以小喇叭是更优解（现场提到小喇叭比大喇叭体积小约 30%）。
<strong>为什么重要</strong>：选喇叭不是比口径，是比"喇叭+腔体"的整体声学，还要结合产品真实用途（人声 vs 音乐）来定。
<strong>你能用</strong>：按"主要发什么声"选喇叭等级；为小喇叭设计够大的腔体，往往比硬塞大喇叭更划算。

### 3. 电池按续航和使用场景倒推，"偏大"是要被砍的
逐字稿里电池被判"偏大"：实测 1200mAh 配选定喇叭可连续约 3 小时，而设备"主要定义为车载、基本都连电线"，1 小时续航就够，于是结论是体积可缩到一半，把省下的空间让给结构。最终还要看外形比例再定 1200mAh 还是更小（提到外形宽度若有 48–50mm，1200mAh 也塞得下）。
<strong>为什么重要</strong>：电池是体积/重量大户，凭"多给点保险"的直觉容易挤死结构空间。正确做法是用真实使用时长 + 是否常插电来倒推容量。
<strong>你能用</strong>：定电池前先问"用户一次用多久、是否常接电"，按场景倒推，别默认堆大容量。

### 4. 屏幕会衰减雷达信号——传感器布局的物理硬约束
一处关键纠错：雷达本想放头部，但头部有屏幕，"玻璃对雷达信号会衰减很大"，于是确认雷达放身体。身体那块因此只需一小块板（不像头部那块还要叠后板做红外数据传输）。
<strong>为什么重要</strong>：这是容易被忽略的物理约束——把无线/雷达传感器藏在屏幕、金属或厚壳后面会显著掉性能。布局阶段必须把"信号穿透"纳入考量。
<strong>你能用</strong>：放雷达/无线/红外模块前，先确认它前方没有屏幕、金属或会衰减信号的材料挡着。

### 5. 外观先占位、结构后对接：省掉"从零重做"的协同顺序
设计方点明工作法："模型已经建好，先把你确定的元器件放进 3D 模型，在外观阶段全放进去，后面到结构再对接，整体就不会大改动；否则整个模型从零做非常麻烦。" 同时所有尺寸以"已定的屏幕（约 46mm 多宽）"为基准反推头身比例。
<strong>为什么重要</strong>：硬件返工成本极高，外观和结构脱节是返工主因。先锁一个不变基准件、把已确定的元器件提前占位，是低成本的协同纪律。
<strong>你能用</strong>：项目里先确定一个"不会再变"的基准件，围着它排布其余元器件，定型前把所有已知约束一次性灌进模型。

### 6. 元器件厂家只给 2D 尺寸图时，自己建模并核对误差
屏幕厂家只提供了 CAD 2D 尺寸图、没给 3D 模型，设计方就按尺寸图自建模型，并明确提示"开模/批量时尺寸都会有点偏差"、需要核对。能拿到厂家 3D（STP）最好，拿不到就用 2D 图核对。
<strong>为什么重要</strong>：上游给料不齐是硬件常态，能不能用 2D 图自建可靠模型、并预留公差，直接决定后面装配会不会出错。
<strong>你能用</strong>：向元器件厂家优先要 3D（STP/STEP）；只有 2D 时自己建模并主动预留装配公差。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：声音问题先排查声学结构，再换料</strong>
- 核心意思：出音口 / 朝向 / 腔体密封没做好，再大的喇叭也白搭。
- 你可以怎么用：声音小时按"出音口→朝向→腔体"顺序排查，最后才考虑换喇叭。
- 不要误读成：声音小=喇叭功率不够——多数情况是结构没留出音口。

<strong>方法 2：元器件选型按真实场景倒推</strong>
- 核心意思：续航看是否常插电、传感器位置看信号会不会被挡，凭场景定参数。
- 你可以怎么用：列出每个元器件的真实工作条件，再定尺寸/容量/位置。
- 不要误读成：参数越大越保险——余量堆过头会挤死结构、抬高成本和体积。

<strong>方法 3：外观先占位、锁基准件、结构后对接</strong>
- 核心意思：先把已确定元器件塞进外观模型、围一个不变基准件排布，再进结构。
- 你可以怎么用：定型前把所有已知约束一次性灌进模型，减少后续大改。
- 不要误读成：等元器件全齐了再开始建模——那样会一直等、也错过定型窗口。

## 📅 7 天作业

- <strong>Day 1</strong>：找一个你手边的智能硬件，拆解它喇叭的出音口在哪、朝向哪，对照它的实际听感。
- <strong>Day 2</strong>：给你正在做/关注的一个硬件，按"真实使用时长 + 是否常插电"重新估一遍电池容量，看现配是否偏大。
- <strong>Day 3</strong>：列出你产品里所有无线/传感器模块，逐个确认前方有没有屏幕/金属/厚壳挡信号。
- <strong>Day 4</strong>：画一张你产品的"基准件"图——哪个件最不可能变（屏幕？主板？），其余件如何围它排布。
- <strong>Day 5</strong>：找一款小喇叭和一款大喇叭的腔体方案对比，理解"腔体>口径"对人声场景意味着什么。
- <strong>Day 6</strong>：跟一个上游元器件厂家要 3D 模型；要不到就拿 2D 尺寸图自己建一遍并标公差。
- <strong>Day 7</strong>：复盘一次你或同行的硬件返工，判断它是不是"外观/结构脱节、没提前占位"造成的。
