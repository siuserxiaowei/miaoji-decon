>

## ⏱ 如果只读 10 分钟（7 条带走判断）

1. <strong>"AI 自我进化"是科幻不是产品</strong>。客户反复想要一个"能自我学习、有意识、越用越懂孩子"的小智，方案商直接定性："这是科幻电影里的东西""模型不会自己训练，都要人为微调"。你想做 AI 产品，先把"自进化"从需求清单里划掉。
2. <strong>客户以为的"记忆/进化"，本质只是"知识库 + 长期记忆 + 总结"三件套</strong>。把对话记录下来、按关键词调取、定期总结提醒——这就是市面上能做到的全部，不神秘也不智能。需求要翻译成这三个工程词，才谈得下去。
3. <strong>小智的真相：ESP32 板子 + 云端大模型 API</strong>。硬件极便宜（低成本交互/采集），所有"智能"都在服务端。"再小的板子也塞不下大模型"，所以<strong>有没有记忆都必须配服务器</strong>。别指望端侧硬件本身变聪明。
4. <strong>国内靠 IP+营销，国外靠功能——这是同一品类的两套打法</strong>。方案商原话："国内市场就是靠 IP 跟营销，拉布布啥功能没有照样卖很贵；国外客户更看重功能。"你做哪个市场，决定你押营销还是押功能。
5. <strong>"虾哥/小智"是一个真人创始人做出来的开源生态，且明确允许商用</strong>。虾哥原本做 K12 赚了钱，"玩着玩着做成生意"，IP 已立。代理协议允许任何人商用，基于 ESP32 二次开发不踩授权坑。这是这条赛道能批量涌入方案商的制度前提。
6. <strong>"红利期越来越窄，要做就尽快做"是方案商的真实判断也是销售话术</strong>——两者同时成立。这话既可能是行业实情，也可能是逼单。带着这个双重性听。
7. <strong>一套定制（采集弹幕/对话→实时回复类）报价六七万、25个工作日</strong>，交付=研发+适配你结构的硬件样品+烧录固件+服务端源码+系统源码。这是当下小智类方案的一个真实价格锚点（单一现场口径，需横向比价）。

## 📚 深挖：7 个值得单独学的知识点

### 1. "AI 自我进化"为什么是伪需求（全场最该带走的一课）
客户三次表达同一个幻想：让小智"有意识""自我进化""越用越懂孩子，能主动深度交流"。方案商每次都摁回去——"现在 AI 没什么意思，都是调知识库""没有自己意识的，进化到那种程度是科幻电影里的东西""模型不会自己训练，都要人为去微调大模型才行"。
<strong>为什么重要</strong>：这是当下 LLM 产品的硬边界。大模型在推理时<strong>不学习</strong>，它不会因为跟某个孩子聊得多就"变懂"这个孩子——它只是在每次对话时把"被检索出来的历史"塞进上下文。把"自进化"当卖点的产品，要么在画饼，要么偷换成了"记忆+总结"。
<strong>你能用</strong>：任何"越用越智能/会自我成长"的 AI 需求，先翻译成"它到底是检索了更多历史，还是真的被重新训练了？"——后者在消费级产品里基本不存在。

### 2. "记忆"的真身：知识库 + 长期记忆 + 总结，三件事拼出来的
客户心里的"记忆"很魔幻（记住孩子特点、分析、给建议、以后提醒）。方案商把它拆成可工程化的部件：
- <strong>知识库</strong>：把"学习内容"做成专门库，对话时检索它来回答——"一问一答""调知识库"。
- <strong>长期记忆</strong>：把对话/事件记录下来、长期保存、按关键词调取（类比 ChatGPT 的记忆功能，但"咱们这个数据可以更大、保存更久"）。
- <strong>总结</strong>：定期把记录归纳成结论，下次类似情况触发提醒——客户描述的"记录→分析→建议→以后提醒"，方案商一句"这只是总结，然后再转起来"。
<strong>为什么重要</strong>：这是"把幻想变成报价单"的标准动作。需求方说不清楚，是因为没有这套词汇；有了它，三件套各自都是成熟模块。
<strong>你能用</strong>：下次提 AI 需求，强迫自己用"知识库/长期记忆/总结"三个词重写一遍，做不出来的部分通常就是科幻部分。

### 3. 小智的架构真相：ESP32 + 云端大模型，端侧只是壳
方案商解释得很清楚：小智基于 <strong>ESP32 电路板</strong>，它"实现了非常低成本的交互/采集功能"，所以整机成本极低；但"再小的板子也没法把整个大模型轻量化塞进去"，<strong>所有智能都在服务端</strong>——"有没有记忆都需要服务端"。官方版小智本地部署、卡多所以响应快；方案商走 API 模式，靠 9月24 新模型把响应做到"1秒左右，基本跟小智没区别"。
<strong>为什么重要</strong>：这决定了成本结构和你能/不能省的地方。硬件可以做到很便宜，但只要你想要"记忆/智能"，服务器（及其持续成本）是逃不掉的固定项。
<strong>你能用</strong>：评估任何 AI 硬件成本时，把账分成"一次性硬件 BOM"和"长期云端/服务器"两本，后者才是规模化后的大头。

### 4. 同一品类的两套商业逻辑：国内 IP+营销 vs 国外功能
方案商的判断很直白："国内市场就是靠 IP 跟营销，不怎么靠功能——你看拉布布，啥功能没有，靠营销卖那么贵。""国外客户会更注重功能上的东西。"还补了一刀：国内深圳/汕头澄海"做了一大堆垃圾在国内卖"，国外没那么卷。
<strong>为什么重要</strong>：这意味着"做国内"和"做国外"几乎是两个生意——投入项（设计师/网红 vs 算法/功能）、壁垒、节奏都不同。客户明确要做美国市场，那就该把资源压在"长期记忆"这种功能差异化上，而不是 IP。
<strong>你能用</strong>：定市场=定打法。先回答"我靠 IP 还是靠功能赚钱"，再决定团队该招营销还是招工程。

### 5. 虾哥与小智生态：一个真人 + 一份允许商用的协议
"虾哥是一个人吗？""对，是一个人，小智生态的创始人。"他本来公司做 K12 赚了很多钱，"玩着玩着把这个做成了一门生意"，现在 IP 已立——"人家说小智就知道是虾哥"。关键的商业前提：<strong>代理人协议里写明允许任何人使用或商用</strong>，所以基于 ESP32 二次开发不会有授权问题。
<strong>为什么重要</strong>：一个赛道能不能"批量涌入方案商、人人下场卷"，制度前提就是授权是否开放。小智的开放协议正是这波 AI 陪伴硬件方案商遍地的底层原因。
<strong>你能用</strong>：进任何开源/生态型赛道前，先把"商用授权条款"读一遍——它决定你是站在巨人肩上还是踩在雷上。（本场为方案商口径，自己务必核实协议原文。）

### 6. 用对标产品反推需求：LOKI / 大疆 / 大麦机
客户描述"摄像头一直记录、AI 总结你的一天"，方案商直接给坐标："你说的这个叫 <strong>LOKI</strong>，融了一千多万，海外卖得很好；竞品还有大疆、（疑似）影石360、各种 AI 眼镜——卖点是全天无间断拍摄、AI 剪辑生成你的一天、忘了做过啥可以问它。"无人直播需求则对标自家做过的"大麦机"（采集抖音弹幕→排序→实时回复，TikTok 逻辑相同未实测）。
<strong>为什么重要</strong>：当需求方说不清，最快的对齐方式不是文字描述，而是"你要的是不是 XX 那款"。一个对标产品 = 一份现成的功能规格 + 商业验证。
<strong>你能用</strong>：提需求时附 1-2 个对标产品名，比写三页 PRD 更高效；调研赛道时，先扒清头部融资额和卖点。

### 7. 一通售前对话的反面教材：需求说不清，拿不到方案
这通话里最戏剧的张力，是方案商反复说"你说的东西太少了/太宽泛了，没法给你做需求"。客户一边自称小白、没规划，一边要方案商给"创意"。方案商的应对很职业：让"胡工截图别的客户需求文档（去掉结构/外壳/客户信息）"给他看，"客户大概知道自己要做什么、哪些模块"。
<strong>为什么重要</strong>：这是真实商业沟通的常态——<strong>模糊的需求换不来认真的方案，只能换来模板和报价</strong>。客户的"启发式提问"对自己有用，但让对方无从发力。
<strong>你能用</strong>：找供应商/外包前，哪怕是小白，也先用"对标产品+三件套功能+目标市场"把需求写成半页纸，对方才接得住、才肯认真出活。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：把"AI 需求"翻译成知识库/长期记忆/总结三件套</strong>
- 核心意思：消费级 AI 能做的就这三件事的组合，把模糊愿望映射进去。
- 你可以怎么用：写需求文档时逐条标注属于哪一件，标不进去的先搁置。
- 不要误读成：以为加了"记忆"就等于 AI 会"懂"用户——它只是检索历史塞进上下文，不是真的学习。

<strong>方法 2：先做软件/网页版 MVP，再上硬件</strong>
- 核心意思：方案商提到智能客服"可以先做个软件或网页"，逻辑和大麦机一样——先验证再投硬件。
- 你可以怎么用：想做陪伴/客服类 AI，先用纯软件跑通对话+记忆+总结，验证用户愿不愿用，再花钱开模做硬件。
- 不要误读成：跳过硬件——而是把硬件放到"软件验证过"之后，降低试错成本。

<strong>方法 3：用对标产品对齐需求与报价</strong>
- 核心意思：LOKI/大麦机这种现成对标，既是功能规格也是商业验证。
- 你可以怎么用：跟供应商沟通时直接说"做成 XX 那样"，比抽象描述快十倍。
- 不要误读成：照抄对标就能卖——对标是对齐语言用的，差异化（如美国市场要功能）还得自己加。

## 📅 7 天作业

- <strong>Day 1</strong>：把你心里的 AI 产品需求，逐条标注属于"知识库 / 长期记忆 / 总结"哪一类，挑出标不进去的"科幻条目"删掉。
- <strong>Day 2</strong>：给你的目标产品找 2 个海外对标（像 LOKI 那样），记录它们的融资额、核心卖点、几个使用模式。
- <strong>Day 3</strong>：算两本账——硬件一次性 BOM 成本 vs 服务端/云端 API 的长期成本，看规模化后哪本更重。
- <strong>Day 4</strong>：明确你做国内还是国外市场，并据此写下"我该把钱压在 IP营销 还是 功能"的一句话结论。
- <strong>Day 5</strong>：用半页纸写一份"对标产品 + 三件套功能 + 目标市场"的需求简报，标准是"陌生方案商看完能直接报价"。
- <strong>Day 6</strong>：去读一遍小智/你所用开源生态的商用授权条款原文，确认能不能商用、有没有坑。
- <strong>Day 7</strong>：用纯软件或网页版搭一个最小对话 Demo（接通义千问或豆包），自测"记忆+总结"到底能做到什么程度。
