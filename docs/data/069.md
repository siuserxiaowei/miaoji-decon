## ⏱ 如果只读 10 分钟（6 条带走判断）

1. <strong>开源小智方案能玩，但不能直接商用</strong>——它没有支付功能，且默认让"小白用户自己去申请各家大模型 API Key"，95%+ 用户根本搞不定。商用必须有人把 API Key 设成"管理员全局配置"，用户无感使用。
2. <strong>"免费大模型"是错觉</strong>——商用调用大模型 API 基本都要付费，所谓免费额度测试阶段就用完了。做硬件前先把这条算进成本，别以为接个大模型不花钱。
3. <strong>AI 玩具是"先出外壳结构，再适配主板"，不是先做硬件</strong>——方案商按你的外壳尺寸去调电路板大小，顺序反了会返工。
4. <strong>淘宝爆款 AI 玩具退货率很高</strong>——"一时兴起买来玩一下没感觉就退了"。这位卖家有现成私域粉丝+团购盘子，反而比纯铺货的更稳，这是他的核心优势。
5. <strong>眼睛屏幕是 AI 玩具体验的命门</strong>——0.7 寸屏普遍 160×160，分辨率低、易有马赛克；而且同样 0.7 寸不同厂商质量差很多。因为眼睛是"买家能直接接触到的东西"，模糊就劝退。
6. <strong>方案商的交付清单要白纸黑字</strong>——固件源代码、Web 管理端源码、小程序源码、通信接口、协议文档全给；报价 5 万、工期约 1 个月、付款 5-5 或 5-3-2、签合同走公对公。

## 📚 深挖：7 个值得单独学的知识点

### 1. 一台 AI 语音玩具的完整技术链路（小白必懂的那张图）
服务商完整讲了一遍流程：开机→检测是否联网→未联网进<strong>配网模式</strong>→（华南理工方案）连 Java 服务器查数据库里有没有这台设备（每台设备有 MAC 地址）→服务器下发<strong>六位验证码</strong>（像手机验证码，但存在服务端）→玩具语音播报验证码→用户在网页填入验证码完成绑定→之后硬件通过 <strong>WebSocket</strong> 连接 Python 服务端→服务端提供能力：<strong>ASR</strong>（语音识别）→<strong>大模型</strong>搜索/对话→<strong>TTS</strong>（语音合成）回复。
<strong>为什么重要</strong>：这是所有 AI 语音硬件的通用骨架，不懂这条链路就无法判断方案商在卖什么、坑在哪。
<strong>你能用</strong>：把"配网→绑定→ASR→大模型→TTS"背下来，对接任何 AI 硬件方案都用得上。

### 2. 开源方案为什么不能直接商用（两条死穴）
服务商反复强调开源小智方案商用的两个硬伤：
- <strong>没有支付功能</strong>——商用要卖"角色/能力/订阅"（如花 9.9 买一个陪伴角色），开源方案没有这层。
- <strong>权限/配网方式很乱</strong>——默认让"小白用户自己去千问等平台申请 API Key"，极其繁琐，"95% 甚至更多用户用不上"。
解法：把 API Key 设成<strong>管理员全局配置</strong>，用户只管用、无需自己申请；商用版需要补支付、补管理后台。
<strong>为什么重要</strong>：这是"开源 Demo 能跑 ≠ 能卖"的经典分水岭，决定要不要自研。
<strong>你能用</strong>：评估任何开源硬件方案时，先问"支付和密钥管理谁来扛"，这两条没解决就不能商用。

### 3. "免费大模型"是错觉，API 调用要算进成本
现场原话："哪有免费大模型，没有免费大模型……基本上都是需要钱的。"所谓免费额度，"测试的时候差不多就把这个额度用完了"。商用必须按量付费买 token。
<strong>为什么重要</strong>：很多新手以为接大模型不花钱，结果商用即亏损。
<strong>你能用</strong>：做 AI 硬件/App 报价前，把每个活跃用户的大模型调用成本算清，再定价。

### 4. 做硬件的正确顺序：外壳结构先行，主板后适配
服务商纠正了一个常见误区——"不是先出硬件，是先出外壳跟结构，我们这些板子再根据你的结构跟外壳去调"。因为电路板大小要迁就外壳空间（这只玩具脑袋约 7cm³、头身比 1:1，身体塞不进东西，所有元件都放脑袋里）。
<strong>为什么重要</strong>：顺序反了（先定主板再做外壳）容易塞不进、返工。
<strong>你能用</strong>：你的硬件项目先冻结外壳/结构尺寸，再让方案商按尺寸定主板。

### 5. 屏幕（眼睛）选型：体验的命门，0.7 寸是重灾区
- 0.7 寸屏普遍是 <strong>160×160</strong> 分辨率，偏低、"明显马赛克"；
- 同样 0.7 寸，不同厂商质量差很多——"有的 0.7 的就比较清楚，有的马赛克很重"；
- 1.28 寸、1.5 寸明显更清楚；现场提到"无名科技"家的眼睛屏比较清晰可作对标（人名/厂名以 ASR 为准，待核）。
- 关键判断："眼睛放在外面、买家能直接接触，配不清楚体验感很差。"
<strong>为什么重要</strong>：眼睛是 AI 玩具最直接的体验触点，屏一糊整个产品掉档。
<strong>你能用</strong>：选屏别只看尺寸，优先 1.28 寸以上，或亲手验过的清晰 0.7 寸；务必拿样品实拍对比。

### 6. 续航与充电底座：待机要扛得住
卖家想要"连续对话四五小时"——服务商直接判断<strong>不现实</strong>，但<strong>总待机时间撑四五小时</strong>可做，方案是 <strong>2000mAh</strong> 电池 + <strong>充电底座</strong>（"一天充十几次电体验感很差，最好带个底座"）。
<strong>为什么重要</strong>：续航预期不切实际会导致设计返工或差评，底座是低成本解法。
<strong>你能用</strong>：给对话类硬件设计电源时，区分"连续对话时长"和"总待机时长"，标配充电底座兜底。

### 7. 方案商的交付清单与付款结构（避免扯皮）
- <strong>交付物</strong>：一整套研发 = Web 端 + 小程序 + 配网 + 一套 3D 打印样品；源码层面给<strong>固件完整源代码、Web 端源码、小程序源码、通信接口、协议文档</strong>。
- <strong>报价/工期</strong>：约 5 万元；因方案商单子多，"要先给老客户做"，预计某日才能开工，做约 1 个月。
- <strong>付款</strong>：<strong>5-5 或 5-3-2</strong>，定金 5 成开工，交付验收后 5-7 天结尾款；<strong>公对公、签合同</strong>（"签合同对双方有保障"），开票齐全。
- <strong>售后</strong>：合同里写——属内容交付问题方案商负责，属客户侧问题不兜。
<strong>为什么重要</strong>：源码版权与付款节点是方案商合作最容易扯皮的地方（参见你库里"方案商版权归甲方"的结构性坑）。
<strong>你能用</strong>：外包硬件方案时，把"全套源码+协议文档+售后边界+付款节点"全部写进合同。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：用"管理员全局配置 API Key"把开源方案商用化</strong>
- 核心意思：别让终端用户自己申请密钥，由管理员在 Web 后台全局配好，用户无感调用。
- 你可以怎么用：做任何 to C 的 AI 硬件/工具，把密钥/配置收到管理后台。
- 不要误读成：直接把开源 Demo 套壳就卖——没解决支付和密钥就是给自己挖坑。

<strong>方法 2：靠私域+复购盘子扛住硬件的高退货率</strong>
- 核心意思：这位卖家有固定粉丝、按团团购（一团一千多个、能开几团），比纯铺货抗风险。
- 你可以怎么用：上硬件前先把私域和复购渠道攒起来，把首批卖给信任你的人。
- 不要误读成：私域=随便就能卖——并发量、复购意愿仍要验证。

<strong>方法 3：第一代只做基础功能，迭代留给二三代</strong>
- 核心意思：第一阶段先做"含全部基础语音功能 + 一个清晰点的眼睛"，小程序等后续再加。
- 你可以怎么用：把 MVP 砍到最小可卖，先上市验证再迭代。
- 不要误读成：第一代就堆满性格养成/记忆/支付——会拖死上市时间。
