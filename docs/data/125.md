## ⏱ 如果只读 10 分钟（8 条带走判断）

1. <strong>订阅电商在美国能成，本质是「卖给不爱上网的人」</strong>。橙酱（美国订阅电商）的用户画像是 banker / 金融从业者、白人、自由职业者——他们不爱网购，就喜欢订完东西每月等上门。生命线是「5 个月」：撑过 5 个月的用户基本能续到 2-3 年。这不是产品好，是<strong>用户结构决定了低流失</strong>。

2. <strong>国内几乎没有订阅土壤，订阅电商基本只能做美国</strong>。现场反复印证：订阅袜子/内裤/食谱/咖啡这类盒子（subscription box），「美国才有，英国也有但太少」，国内只有「男人袜」勉强成立。变现模式必须分国别设计。

3. <strong>方案商的真相：本质是外包，赚的是信任成本和爆品分成</strong>。小伟原话——「那么多家方案商，为什么要在你家做这一款产品？基本上每个大单都要过好几个月的信任成本。」前期垫钱不多，但靠的是熬出 1-2 个爆品拿销售分成。版权/归属是结构性风险。

4. <strong>API 中转站是一门「劫富济贫 + 压钱」的生意，护城河是速度和稳定</strong>。彭涛（自称圈内前五/前十）揭底：中转站「便宜、稳定、速度快」三者不可能全得；高客单用官方正规号，低客单才用逆向；要做好体验得「压个百万」级别的资金、服务器月费上万。速度比头部快 3 倍靠的是网络优化。

5. <strong>健康硬件的留存密码不是「记录」，是「预测风险」</strong>。做健康追踪仪/扫脸/智能戒指的几家共识：给用户看已有体检数据=看完就走（像看一次体检报告）；只有「预测你未来可能生什么病」才有持续使用动力。一个反面案例：用户被预测出风险、去医院查没事，反过来要求赔医药费。

6. <strong>苹果生态太贵，灰产和增长玩家都在往 Web 端逃</strong>。Adaptive 的 Web-to-App 付费墙、橙酱用 PWA（小程序式）替代上架 App、龙虾盒子/工具线干脆「包都不发商店、纯做 web 端」——避开 App Store 审核、隐私合规和 30% 抽成，是一条系统性的躲避路径。

7. <strong>ToDesk「没有竞品」的真相：付费率极低 + 老板不怕卷</strong>。年营收约 2 亿、95% 来自 C 端、覆盖率 60%，但付费率很低（人人想要免费替代品/会用 RustDesk）。它的「护城河」其实是分场景精细化收费的套路 + 老板（YY/PIGO 系，老胡）不在乎骂声也不怕被卷。

8. <strong>出海税务与政策恐惧正在把公司主体推向新加坡</strong>。多位把主体迁新加坡（电商「各种问题」、医疗房被「大巴拉走抓人」、白冰被追补税的故事），核心是<strong>国内政策不确定性</strong>——「今年这样，明年就变」。但新加坡成本贵，常见解法：主体新加坡 + 研发运营在泰国/越南。

## 🎯 这篇真正能学什么

这场饭局练的不是「记住谁卖了什么」，而是三种眼力：①<strong>看穿一门生意的真实商业模式</strong>——订阅电商赚的是用户结构、方案商赚的是信任+分成、中转站赚的是速度+压钱、健康硬件赚的是「预测」而非「记录」；②<strong>识别出海玩家的系统性套路</strong>——往 Web 端逃避苹果生态、把主体迁新加坡逃避政策、用逆向/矩阵号薅羊毛；③<strong>理解「情绪价值/玄学」为什么能定价</strong>——算命、圣经、AI 陪伴、宠物翻译，底层都是把心理刚需做成可订阅的产品。读完别记产品名，记这几张「生意结构图」。

## 📚 深挖：10 个值得单独学的知识点

### 1. 订阅电商的命门是「用户结构」，不是产品
橙酱（美国订阅电商，做袜子/内裤/食谱式订阅盒）反复强调：核心用户是 <strong>banker、金融从业者、白人、自由职业者——「不爱上网、不爱网购」的人</strong>。他们订阅的动机就是「尽量减少跟平台交互的频率，每个月等东西上门就好」。所以退订率天然低，「生命线」是 5 个月：撑过 5 个月的用户基本能续 2-3 年；5 个月之前不续的「基本就不是我的用户了」。今年付费订阅用户约 2-3 万。
<strong>为什么重要</strong>：这颠覆了「订阅靠产品体验留人」的直觉。真正决定留存的是<strong>你选的用户本来就不流失</strong>。同理可推：美国男性平均每人有 4 个订阅（次数多金额小），女性 2-3 个（次数少金额高）。
<strong>你能用</strong>：做订阅前先问「我的目标用户是天然高频换还是天然懒得动？」——选对人群比优化产品更重要。

### 2. 国内没有订阅土壤——订阅电商基本=美国生意
现场多次印证：subscription box（订花、订牛奶、订袜子、订内裤、订食谱、戒酒寄书、彩妆小样）「美国才有，英国也有但太少，因为获客成本」。国内能成的几乎只有「男人袜」。还有「坏男孩俱乐部」（寄食谱）、戒酒俱乐部（每月寄书+线下互助会）这类把「行为改变」做成订阅的玩法。
<strong>为什么重要</strong>：印证了 AI 硬件场那篇的判断——「中国人没有订阅习惯」。这是变现模式的硬约束，不是运营能补的。
<strong>你能用</strong>：任何订阅模型先确认主战场是不是海外；别把美国订阅逻辑硬套国内。

### 3. 方案商的真相：本质是外包，赚信任成本 + 爆品分成
小伟（记录者本人）自述得最透：做 AI 语音对话机器人方案商=代工厂，「甲方给钱我们做研发，他负责销售和渠道」。前期垫钱不多，但<strong>做 B 端获客「每个大单都要过好几个月的信任成本」</strong>——那么多方案商，凭什么选你？「本质还是外包，但如果能做出 1-2 个爆品就比较赚」，靠每卖一台分成。团队配置：固件、硬件结构、服务端、财务，5 个核心 + 十几人。
<strong>为什么重要</strong>：方案商看似有产品，其实卖的是「被信任的研发能力」，且版权/成果归甲方（AI 硬件场那篇有方案商预售 800 台被甲方换人、前功尽弃的血案）。
<strong>你能用</strong>：做方案商要把信任成本前置（案例/口碑/试做）当成获客主战场；想做长久价值就得攒自有品牌或爆品分成。

### 4. API 中转站：一门「便宜/稳定/快不可能全得」的压钱生意
彭涛（程序员出身，创业第六年，自称圈内前五/前十，做了十来个网站，现主做中转站，「跟特朗普做同样的项目」的玩笑）揭底中转站的真实结构：
- <strong>三角不可能</strong>：便宜、稳定、速度快，三者不可兼得。
- <strong>分层供给</strong>：低客单用户用逆向（Claude/Grok/各种逆向，时断时续）；高客单（3 块、5 折这种）用<strong>官方正规号、甚至官方 API、AWS 大量采购</strong>，稳定不用担心。
- <strong>要做好得压钱</strong>：「大概压个百万」级别资金，服务器月费上万；速度比头部快 3 倍靠网络优化。
- <strong>薅羊毛是圈内共识</strong>：用一堆号矩阵薅羊毛，或做「映射」——同样调 GPT/Solid，实际映射到便宜库上，价格能差 5 倍。被戏称「劫富济贫」。红线是：逆向能力绝对不能在社媒公开宣传。
<strong>为什么重要</strong>：这是 AI 应用底层「卖 token/算力」生意的真实毛利结构和风控逻辑。
<strong>你能用</strong>：内部企业用可以自己搭中转（没技术含量）；但要做几千人在线的商业级，拼的是资金、服务器、网络优化——别被「人人能搭」误导。

### 5. 健康硬件的留存密码：把「记录」升级成「预测」
做健康追踪仪/AI 扫脸/智能戒指的几位共识极其一致：
- 「已有身体数据对用户没多大意义，像看一次体检报告就走了，没有持续动力。」
- <strong>「如果能帮他预测未来可能产生什么风险、可能会生病，用户就有持续使用的动力。」</strong>
- AI 扫脸原理：脸部表皮最薄，通过反光测心率、血压等（被称「抖音金主网红」产品，曾月收入 400 万）。
- 智能戒指（Blast 即将推出，保密状态）、宠物健康/情绪监测（全天候记录宠物体温变化预测生病）都走同一逻辑。
- ⚠️ 反面案例：有用户被测出风险、去医院查无事，反过来要求方赔偿医药费——预测类产品有「假阳性」责任风险。
<strong>为什么重要</strong>：这是所有「数据类硬件」的通用留存公式：<strong>记录=一次性，预测=持续性</strong>。
<strong>你能用</strong>：做任何记录/监测类产品，把交付物从「展示历史」改成「预测未来/给出行动建议」，留存逻辑完全不同；但要管理预测误差带来的责任风险。

### 6. 苹果生态太贵→集体往 Web 端逃（PWA / Web-to-App）
多条线索拼成一个系统性趋势：
- 橙酱放弃做 App，改用 <strong>PWA</strong>：用 Shopify 在页面上生成 PWA 即可，「小程序涉及的东西更麻烦」，不用上架、不受应用商店规范限制、避开合规审查。
- Adaptive 的第二条业务线就是 <strong>Web-to-App / H5 付费墙</strong>：iOS 生态贵，很多公司（含「灰产老产业」）转移到 Web 端获客，先在 Web 完成定位流程/付费墙，再引导下载 App。健康、教育、AI 生成类走这条路多。GlennAI 等也是「下载量高、付费墙高，但更大收入在外端（Web）」。
- 工具线/龙虾盒子干脆「有些包都不往商店发，纯做 web 端，就不受限制」。
<strong>为什么重要</strong>：30% 抽成 + 审核 + 隐私合规三座大山，催生了「绕过 App Store」这条系统性增长路径。
<strong>你能用</strong>：做订阅/内容/AI 产品，认真评估 PWA 或 Web 端付费墙作为主获客/变现入口，尤其当苹果抽成吃掉你大部分利润时。

### 7. 付费墙 SaaS 的核心价值：无限 A/B + 数据预测，不是「墙」本身
谷歌系/Adaptive 那位讲得很清楚：付费墙页面本身简单，「核心能力在基于付费墙的其他方面」：
- <strong>自研付费墙要提交应用商店审核才能上线；用他们的平台不用，可做无限制 A/B 测试</strong>跑出转化率最高方案。
- 与 MMP/分析平台打通，基于数据积累做<strong>预测</strong>：某方案未来半年/一年能带来多少收益，辅助财务、营收、买量决策。
- 直接竞品是 <strong>RevenueCat</strong>（开发者关系做得极大，主打服务一人公司/个人开发者，让他们把研发精力放产品而非这些「边角料」）。Adaptive 自称国内目前只有他们一家进来。
<strong>为什么重要</strong>：这是「卖铲子」生意的范本——价值不在功能本身（墙谁都会做），而在<strong>绕过审核的 A/B 速度 + 跨产品数据沉淀</strong>。
<strong>你能用</strong>：做工具/SaaS 时，问自己真正的护城河是不是「数据复利 + 帮客户省掉某个昂贵流程」，而非单点功能。

### 8. 「情绪价值/玄学」为什么能定价：把心理刚需做成可订阅产品
满桌都在做「情绪/心理」外壳的产品，逻辑高度一致：
- <strong>纸条 App（陈立军/Flomo 系笔记）</strong>：核心不是帮你整理笔记（那 AI 都能做），而是给一个小 AI 角色（参考蔚来 NOMI），每天记录后给你一句名人名言、「心情不好时它不想理你」的人性化设计——把枯燥的记录过程加情绪价值。受 Flomo 启发，用微信小程序丢链接即可记录。从星座 App 起家（发现「蛤蟆的情绪」这类词是百万级流量）。
- <strong>算命/玄学出海</strong>：face tell（上海/台湾团队，徐同）做基于中国本土风水命理的占卜，「后面有算法会尽量推积极信息」——本质是情绪价值。难点是命理专业术语的英文本地化「非常复杂漫长」。
- <strong>圣经 App（Blast）</strong>：给美国基督徒做圣经阅读+AI+直播，宗教类排行榜长期前五。延伸出韩国「每月捐 11% 给牧师、牧师不缴税」的高 ARPU 信仰经济观察。
<strong>为什么重要</strong>：玄学/宗教/情绪不是迷信，是<strong>把「安心感」做成可定价、可订阅的产品</strong>；但出海最大的坑是文化本地化。
<strong>你能用</strong>：给工具类产品叠一层「情绪价值/陪伴人格」做差异化；但若涉及命理/宗教，本地化成本要提前算进去。

### 9. 出海税务/政策恐惧，正在把公司主体推向新加坡
反复出现的「出逃」叙事：
- 多位把公司主体迁新加坡：电商「各种问题（不只税）」、政策不确定性。
- 政策恐怖故事：广州番禺万达/科韵路「直接拿大巴过去抓人」（医疗房/灰产）；网红白冰前两年符合减税、做大后被追「重新算、你要补」；「政府 KPI 没完成就找几只肥羊」。
- 共识解法：<strong>主体开在美国/新加坡，研发运营放泰国/越南</strong>（成本低）；新加坡公司有「100-300 天驻留」要求，常派一个人长期驻守。
<strong>为什么重要</strong>：这是出海合规的真实驱动力——<strong>逃的不是税率，是不确定性</strong>。
<strong>你能用</strong>：做海外为主的生意，早期就规划主体架构（新加坡/美国主体 + 东南亚执行），别等做大被盯上才动。

### 10. 区域市场的「反直觉」用户画像（韩国/印度/东南亚）
饭桌上散落一堆一线渠道观察，价值在「反常识」：
- <strong>韩国家庭购物（home shopping，老年向电商）</strong>：买洗衣粉/清洁用品的不是婆婆妈妈，<strong>是大叔</strong>——「韩国大叔很爱干净、爱洗衣服」。
- <strong>韩国 Pulse</strong>：国民级支付 App（类支付宝），近期上线迷你 App + 游戏板块（类微信小游戏），正招甲方入驻，低成本高曝光、「全民会看」，品牌层面划算——出海韩国的新低成本增量渠道。
- <strong>印度</strong>：「难得肯在 1V1（社交/陪聊）上花钱」，因为真人替代品更贵；除了「荷尔蒙的钱」，其他分文不拔。DAU 可以做得极高但 ARPU 低。
- <strong>东南亚用工</strong>：泰国请一个华人要先请一个泰国人（政策）；多请不同国籍反而本地化更好；新加坡/马来人「会搞办公室政治」但也最肯卷（总部常设新加坡因为「只有新加坡人愿意卷」）。
- <strong>台湾</strong>：付费不弱，本土 App 有 17Live（直播）、博弈类多；美图类 Snow/Epic/Soda 受欢迎。
<strong>为什么重要</strong>：每个市场的真实买家画像都和「想当然」不同，错判画像=投放全废。
<strong>你能用</strong>：进任何新市场前，找一个本地一线渠道人验证「到底谁在买、为什么买」，别用国内经验外推。

## 🛠 可直接复用的方法（含「不要误读成」）

<strong>方法 1：用「生命线阈值」判断订阅用户质量</strong>
- 核心意思：橙酱定了「5 个月生命线」——撑过就能续 2-3 年，没撑过基本不是你的用户。
- 你可以怎么用：给你的订阅/续费产品找到那个「拐点月份」，把运营资源集中在帮用户跨过它。
- 不要误读成：所有产品都是 5 个月——这个阈值要你自己用数据跑出来。

<strong>方法 2：把「记录型」产品升级成「预测型」</strong>
- 核心意思：展示历史数据=一次性消费；预测未来风险/给行动建议=持续动力。
- 你可以怎么用：在你的数据产品里加一层「预测/预警/下一步建议」。
- 不要误读成：随便瞎预测——要管理假阳性的信任和责任风险（有人被吓去医院查无事来索赔）。

<strong>方法 3：评估 Web 端（PWA/付费墙）作为主入口</strong>
- 核心意思：苹果 30% 抽成 + 审核 + 合规，催生了「先在 Web 完成付费/定位、再引导下载」的系统打法。
- 你可以怎么用：算清楚苹果抽成吃掉多少利润，测一版 Shopify+PWA 或 H5 付费墙。
- 不要误读成：Web 端=放弃 App——很多是「Web 获客付费 + App 承接体验」的组合，不是二选一。

<strong>方法 4：给硬件/活动准备一个「社交货币」小物</strong>
- 核心意思：小伟用拓竹 3D 打印机几小时打一批 NFC 算命挂件（扫一下出每日一卦），成本极低却成了全场破冰话题和换微信的由头。
- 你可以怎么用：去任何局/展会前，做一个「现场能玩、能算、能拍、能聊」的轻物料。
- 不要误读成：靠小礼物就能成事——它是破冰器，真正的价值还是你自己的能力和后续跟进。

## 📅 7 天作业

- <strong>Day 1</strong>：写下你产品的「持续使用动力」到底是什么（情绪价值/预测/订阅结构/习惯）——如果写不出，说明留存有根本问题。
- <strong>Day 2</strong>：给你的订阅或续费数据找「生命线月份」，看跨过它的用户续费率是不是断崖式提升。
- <strong>Day 3</strong>：如果你做记录/数据类产品，设计一个「预测/预警/下一步建议」功能原型，并想清楚误判的责任边界。
- <strong>Day 4</strong>：算一笔账——苹果 30% 抽成 + 审核周期，对你的利润和迭代速度影响多大？测一版 PWA/Web 付费墙的可行性。
- <strong>Day 5</strong>：如果你是方案商/外包，列出你的「信任成本」获客资产（案例、口碑、试做），缺哪些补哪些。
- <strong>Day 6</strong>：选一个目标海外市场，找一条一线渠道线索，验证「真实买家画像」是否和你的想当然不同。
- <strong>Day 7</strong>：评估你的公司主体/税务架构——若以海外为主，画一版「主体（新加坡/美国）+ 执行（东南亚）」结构图，列出第一步动作。
