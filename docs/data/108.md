>
> ⚠️ 这场会名义上是「合盘谈项目」，真正的硬核其实是两段：①一套<strong>用人话包装的拉人入伙术</strong>（怎么用「心腹/信任」给一个能力不强的人安排低压角色）；②一套<strong>AI 视频投放工厂的产品结构</strong>（品牌库/规则库/生产库三层 + 全接第三方模型）。听的时候要把「话术」和「项目逻辑」分开看。

## ⏱ 如果只读 10 分钟（6 条带走判断）

1. <strong>「心腹」是一种角色设计，不是夸奖</strong>。宋老师反复给小伟定位「你是我的眼睛/心腹」，核心目的不是抬举，是<strong>用低压力、低 KPI 的角色把一个能力不顶尖但忠诚的人绑在广州分部盯人盯钱</strong>。听到有人给你「不要求业绩、只要忠诚」的定位，先想清楚对方在解决他的什么问题。
2. <strong>拉人话术的标准三件套：降压力 + 抬信任 + 讲风险对比</strong>。「你没太大压力」「这队伍里我只信你和强哥」「我还做黄金趋势/收费咨询那些才有风险，这个项目没风险」——三句话分别消解能力焦虑、给情感绑定、用别的高风险业务反衬本项目「靠谱」。
3. <strong>「靠谱」的论证基础值得警惕</strong>：宋老师说这是手上「最靠谱的项目」，依据是「通过命理塔罗 + 看局势」判断的。把玄学占卜当作项目可行性的判断依据，是这场会一个明显的<strong>红旗信号</strong>，听者要自己独立核实商业逻辑。
4. <strong>AI 视频投放工厂的产品结构 = 三层资产库</strong>：品牌库（AI 能理解的品牌感知规则）+ 规则库（营销 SOP/style 写成模型能执行的规则）+ 生产库（批量生成高命中率视频）。<strong>自己只建这三个库，生成全部接第三方模型</strong>（Nano Banana / Sora 等），不做原生模型、不做 LoRA。这是「不套壳但套路」的轻资产打法。
5. <strong>这套工厂赌的是「下一代模型 + 平台广告流深度融合」的时间窗口</strong>。宋老师认为下一代模型会强统一多模态、并高度适配 YouTube 广告流，所以现在先做「规则库 + 感知系统」卡位，等模型成熟时高度匹配。这是个<strong>对未来的下注，不是已验证的事实</strong>。
6. <strong>小伟的真实生意是硬件方案商，刚从亏钱转到赚钱</strong>。挖了两个干了十年的老工程师，定制单接得顺手，前几单亏、现在开始赚，两个客户 6 月量产拿佣金。他对宋老师的项目<strong>没接、也没想好要不要合作</strong>——先把量产单做好再说，是这场会里最清醒的姿态。

## 🎯 这篇真正能学什么

## 📚 深挖：5 个值得单独学的知识点

### 1. 「心腹」定位术：怎么给一个能力不顶尖的人安排角色还让他舒服
现场宋老师把话「摊开讲」：你的定位是心腹，意思是「帮一号位盯住广州分部那些不确定因素，你是我的眼睛」。他特意说<strong>最低要求不是你贡献多少销售</strong>，本质是心腹。配套话术：「这样你没太大压力」「你也在成长」「以前没赚钱的、不靠谱的事我都不想让你参与」。
- <strong>为什么重要</strong>：这是一套完整的「角色 + 情感」打包术。一号位真实需求是「在自己不能常驻的分部，有个忠诚的人盯人盯钱」，但他没说「我需要一个看门人」，而是包装成「你是我最信任的人」。对方既得到了情感满足，又接受了一个明确的功能定位。
- <strong>你能用</strong>：当你需要一个长期、忠诚但能力不必最强的搭子时，给他一个「被信任 + 低 KPI」的清晰定位会比画大饼更稳。反过来，<strong>当你是被定位的那一方</strong>，要分清「他在夸我」和「他在给我派活」——这两件事可以同时发生。

### 2. 拉人话术三件套：降压力 / 抬信任 / 风险对比
把宋老师的话拆开，是一套可复用的说服结构：
- <strong>降压力</strong>：「你没太大压力」「不要求你到岗、不干涉你做别的、你有子公司有基本劳务」——先把对方的入伙成本和风险讲到最低。
- <strong>抬信任稀缺性</strong>：「我对别人天然不信任，现在都是怀疑的」「这队伍里只有你和强哥我真正信任」「只有你配得上这个词」——用「我谁都不信，唯独信你」制造稀缺感和情感绑定。
- <strong>风险对比</strong>：「我还做黄金趋势交易、收费咨询，那些才有大风险，这个项目没什么风险」——拿自己别的高风险业务当背景板，反衬本项目「安全」。
- <strong>为什么重要</strong>：这三招几乎是所有「拉人入伙/合伙」谈话的通用模板。认出它，你就不会被语言本身带着走，而能去问真问题：这项目到底怎么赚钱？我具体干什么？退出条件是什么？
- <strong>你能用</strong>：下次听到「我谁都不信就信你」时，把它当作一个<strong>话术信号</strong>而非事实，然后冷静地问商业细节。

### 3. AI 视频投放工厂的三层资产架构（品牌库 / 规则库 / 生产库）
这是项目的产品内核。宋老师讲「我们不做原生（模型），所有生成都接第三方」，自建的资产是三层：
- <strong>品牌库 / 感知系统</strong>：让 AI 能理解的品牌感知与洞察规则，通过爬虫/关键页抓取把营销结果「软接入」（从投放平台导入到感知系统），「成败在于感知系统」。
- <strong>规则库 / 生产系统</strong>：把营销 SOP 和 style 写成「模型能理解的规则体系」，让创意能被批量执行。
- <strong>生产库</strong>：批量生成视频，追求高完播率/高成交「命中率」和生成一次性（少返工）。
- <strong>为什么重要</strong>：这是一个典型的<strong>轻资产 AI 应用思路</strong>——不碰快速贬值的模型层，只沉淀「品牌理解 + 营销规则 + 生产流程」这类能复用的私有资产。所谓「不是套壳，是套路」「视频版的 Manus（逐字稿写成 minus）」，说的就是用编排和规则在第三方模型上跑出结果。
- <strong>你能用</strong>：做任何 AI 内容/投放工具，先问「我自己沉淀的资产是什么」。如果答案只有「调了某个模型的 API」，那就真的只是套壳；要把领域 know-how（品牌调性、营销 SOP）固化成规则才有壁垒。⚠️ 但这套架构在本场只停留在「讲架构」层面，是否跑得通、命中率到底多少，没有数据支撑，需独立核实。

### 4. 「下一代模型 + 平台广告流深度融合」这个下注，到底赌的是什么
宋老师讲「三个紧迫性」中最核心的一条：下一代模型（逐字稿作「vivo four / view for」，疑似指某下一代多模态模型）会<strong>强统一多模态</strong>，并<strong>高度适配平台广告流</strong>（逐字稿提 YouTube/千川），图文互转的一致性约束更强、规则库更好写入。所以现在先做「前期试水」，后续高度匹配。
- <strong>为什么重要</strong>：这是整个项目的「势」——它不是赌当下能赚钱，而是赌「模型与广告投放深度融合」的窗口期会到来，现在卡位规则库/感知系统，等窗口打开就接得上。这种「为未兑现的趋势提前布局」是常见的早期项目叙事，<strong>可能对、也可能是给现在讲不清的商业模式找的未来理由</strong>。
- <strong>你能用</strong>：评估任何「卡位未来」的项目，区分两件事——「现在已经能赚到的钱（已验证）」和「等趋势来了能赚的钱（下注）」。健康的早期项目应该有一点点已验证的现金流托底，而不是全靠未来。本场宋老师确实提了「已接了一些测试订单」，但金额和真实性未知。

### 5. 小伟的方案商生意：从亏钱到赚钱的真实拐点
后半程小伟讲自己的盘子（这部分信息最实、最可信）：做硬件<strong>方案商</strong>，帮甲方做硬件 + 固件，挖了<strong>两个干了十年的老工程师</strong>，定制单接得顺手；<strong>前几单在亏钱，现在开始赚钱</strong>；有两个客户预计今年 6 月量产，他们负责销售渠道、小伟负责生产拿佣金。一个客户是京东出来的上海交大毕业生（自己做服务端、外包硬件固件），另一个全程交给小伟做、自己只做营销渠道。背后有个苏州大老板投了这两个客户，并暗示「有好 idea 也可以投资小伟」。
- <strong>为什么重要</strong>：这是这场会唯一有<strong>真实经营数据轮廓</strong>的部分——团队（2 个资深工程师）、客户（2 个、6 月量产）、财务拐点（前亏后赚）。对照前半段那个「靠玄学背书、只有架构没有数据」的视频项目，高下立判：<strong>一个是脚踏实地接单跑通的小生意，一个是话术饱满但未验证的大叙事</strong>。
- <strong>你能用</strong>：小伟的应对是范本——面对一号位的拉人邀约，他明确说「我没什么好 idea，也没想好要不要合作，先把量产单做好再说」。<strong>先守住自己已验证的现金流，再考虑别人画的饼</strong>，是早期创业者最该有的纪律。

## 🛠 可直接复用的方法（含「不要误读成」）

<strong>方法 1：用「低压力 + 被信任」给长期搭子定位</strong>
- 核心意思：拉一个忠诚的人长期协作，给他「不背 KPI、但被高度信任」的清晰角色，比画大饼稳。
- 你可以怎么用：明确告诉对方他的职责边界和你为什么选他，让他有安全感。
- 不要误读成：用「信任」当幌子让对方无偿盯活——角色再暖也要有真实的利益安排（小伟这边至少有子公司/基本劳务）。

<strong>方法 2：AI 内容工厂只沉淀私有资产、生成全外包</strong>
- 核心意思：不碰快速贬值的模型层，自建品牌库/规则库/生产库这类能复用的 know-how。
- 你可以怎么用：把你领域的营销 SOP、品牌调性写成模型能执行的规则，配实拍素材提命中率。
- 不要误读成：「接个第三方 API 就是工厂」——没有沉淀私有规则资产的，就是纯套壳。

<strong>方法 3：先用真实小订单验证，再谈融资和放量</strong>
- 核心意思：小伟「前几单亏、现在赚、6 月量产再看」是健康路径；宋老师「接测试订单 + 政府拿钱 + 资本融资」也强调先跑测试。
- 你可以怎么用：任何新项目先跑通一笔真实交易，拿到真实命中率/利润数据再扩张。
- 不要误读成：拿到测试订单就等于模式成立——测试单常是赔本赚吆喝，要看可复制的真实利润。
