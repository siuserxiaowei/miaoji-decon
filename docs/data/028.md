5. <strong>户外可靠性（防潮/高温）在方案商眼里是"打胶防水 + 选耐温芯片"的常规活，不必焦虑</strong>。乙方："打胶就防水了，芯片七八十度都正常。"甲方把它当大难题，其实是已解决的标准工艺。

## 📚 深挖：7 个值得单独学的知识点

### 1. 屏尺寸是硬件方案的第一道生死线
乙方最硬的一条专业判断：<strong>7-8 寸以下小屏随便做；21 寸大屏做不了</strong>。原因不是工艺难，而是<strong>供应链</strong>——大屏要专用显示芯片（提到 MTK、瑞芯微/类似品牌，逐字稿"子宫展位"应为芯片厂商名），这类芯片"市面上买不到，除非你是显示器/投影仪行业的主力客户"。
<strong>为什么重要</strong>：很多小白以为"屏大点小点只是钱多钱少"，其实大屏会直接把整个方案踢出某类方案商的能力圈。屏尺寸决定了你能找哪一类供应商。
<strong>你能用</strong>：硬件方案的第一个问题永远是"屏多大"。要大屏，直接去找做显示器/投影仪/广告机出身的厂商，别找做消费类小屏的。

### 2. 硬件项目的交付边界：板 / 壳 / 量产 / 软件 是四摊事
乙方全程在划边界："我只提供电路板""外壳我报不出来，要单独站""软件你们自己做我们做板"。一个完整硬件＝电路板（方案商）＋外壳结构（设计师/结构厂）＋量产（3D 打印或开模工厂）＋软件（甲方自有团队）。
<strong>为什么重要</strong>：小白最容易的误区是"找一个人把整个东西做出来"。现实是硬件链条天然分工，方案商只接其中一段。打包思维会让你既找错人、又谈不拢价。
<strong>你能用</strong>：立项时画一张"四象限"——板、壳、量产、软件，每格写清楚谁负责、谁出钱、谁担风险。

### 3. 3D 打印 vs 开模：量是唯一的决策变量
乙方给了清晰的成本算术：
- <strong>3D 打印</strong>：单件十几到二十块，无前期开模费，适合几十个的小批量、打样阶段。
- <strong>开模</strong>：开模费一次性约 2-3 万；摊开后单件外壳只要两三块；适合"一个月几千个"的量产。
临界判断："量不够多，开模没有意义。"
<strong>为什么重要</strong>：这是硬件成本结构里最常见、最反直觉的一道题——单件便宜的方案（开模）前期反而最贵。不算总量就拍方案，要么浪费要么挨宰。
<strong>你能用</strong>：任何要做外壳的硬件，先估总量再选工艺：百件级用 3D 打印，千件/月级再考虑开模。

### 4. 户外可靠性 = "打胶防水 + 选耐温芯片"，是常规活不是难题
甲方反复焦虑"云南植物园湿度高、温度变化大，电路板放户外行不行"。乙方的回答把它瞬间降维成标准工艺：<strong>潮湿→给电路板打胶（灌胶）做防水；高温→普通芯片工作温度七八十度，户外不可能超，没问题</strong>。
<strong>为什么重要</strong>：采购方常把"户外可靠性"想象成需要特殊高端方案的大坎，于是溢价采购或迟迟不敢上。其实在有经验的方案商眼里是成熟工艺。识别"哪些焦虑是真问题、哪些是已解决的常规活"，能省钱也能加速决策。
<strong>你能用</strong>：遇到供应商把你的"大难题"轻描淡写解决掉时，别急着不信——可以追问工艺名（如"打胶/灌封"），用术语去交叉验证它是不是行业标配。

### 5. NFC 手环作为低成本"刷卡介质"的分层设计
甲方的打卡介质是手环，乙方点出可以<strong>按成本分层</strong>：
- <strong>最低配</strong>：纯感应（内置 NFC/RFID 芯片，靠近读卡器即触发），"原车里套个卡就行"，无电路、无电、最便宜。
- <strong>高配</strong>：手环带显示屏/灯光，成本显著上升。
读卡器（打卡机）端只要能识别手环里的卡即可。本质是"一对多"——多个手环对一个打卡机。
<strong>为什么重要</strong>：很多需求方一上来就想要"会发光、有屏"的炫酷手环，但 90% 的打卡场景只需要最低配的纯感应。先想清楚"是否真的需要显示"，能砍掉一大块成本。
<strong>你能用</strong>：任何"刷卡/打卡/感应"场景，先问自己"用户端介质到底要不要反馈（灯/屏/声）"——不需要就用纯 NFC，成本最低。

### 6. 户外联网靠 4G 模块，不能指望 WiFi
乙方一句点破：<strong>户外打卡机要联网就得带 4G 模块</strong>，因为户外没有 WiFi 可连。（甲方提到"上海物联网"应为 ASR 误识，语境是"用物联网卡/4G"。）打卡机要做"结算和呈现"就必须实时联网上传，所以这条不能省。
<strong>为什么重要</strong>：联网方式是户外硬件的隐性成本和隐性故障点（4G 模块 + 物联网卡月租）。规划时漏掉它，后期会发现设备根本传不了数据。
<strong>你能用</strong>：任何户外/移动场景的联网硬件，默认预算里要含 4G 模块 + 物联网卡资费，别假设有 WiFi。

### 7. 公模（标准壳）vs 非标开模：能用现成壳就别定制
甲方反复想找"标准化壳子"（公模）省钱，乙方的现实回答："每个产品外壳都不一样，没有刚好适合你内部结构的公模。"但也给了务实折中：<strong>如果对外观要求不高，买个能装下、固定好的现成亚克力盒/标准壳直接塞进去也行</strong>——尤其户外设备只要"是个方盒子"就够。
<strong>为什么重要</strong>：定制外壳（开模/3D 打印 + 结构设计）是隐性大头成本。识别"哪些设备值得做造型、哪些塞进标准壳就行"，是控制硬件成本的关键取舍。本场结论：户外那几个打卡点对造型要求低，可走标准壳；只有少数需要异形的才开模找广告公司/造型厂。
<strong>你能用</strong>：硬件外壳先问"用户会看它、拍它吗"——不会，就用现成标准壳；只有展示面/品牌面才值得花钱定制造型。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：用"场景故事"反推参数，而不是甩参数清单</strong>
- 核心意思：甲方讲清"大巴上发卡 → 园区六个点打卡 → 回车结算呈现"的完整使用流，乙方自己就把"6+1 套设备、屏多大、要不要联网"推出来了。
- 你可以怎么用：跟不熟领域的供应商沟通时，先把完整使用场景讲一遍，让专业方帮你补全技术参数。
- 不要误读成：以为不用做功课就行——你仍要听懂对方反推出的参数（屏尺寸/联网/模块）并复述确认，否则会被带偏。

<strong>方法 2：先要案例对标，再谈报价</strong>
- 核心意思："给成都动物园做过类似的"一句话就建立了信任，比任何报价都管用。
- 你可以怎么用：选供应商时，第一轮先问"做过哪个同类项目，发张照片/案例"，用案例筛掉不靠谱的，再进入报价。
- 不要误读成：有案例就等于便宜或一定合适——案例是信任门槛，价格和合同条款仍要单独谈。

<strong>方法 3：把"快报价"压成"明天上午一个范围"，但接受是估值不是定价</strong>
- 核心意思：甲方要"越快越好"，乙方承诺"明天上午给个范围（几千、不超过 1 万）"，双方都默认这是粗估、不是最终报价。
- 你可以怎么用：首聊就要一个数量级范围（order of magnitude），用来判断"这事大概多少钱、值不值得继续"，不必等精确报价。
- 不要误读成：把"几千不超过 1 万"当成最终成交价——它明确排除了 21 寸大屏、外壳、量产等变量。
