## 📚 深挖：6 个值得单独学的知识点

### 1. "真人数字人陪伴机"是什么，怎么落到一块板子上
讲者说：屏幕上是"真人数字人"，一开始用 MP4（预渲染视频），后来上平台做得"更逼真"。产品方向是医疗、康养、老年陪伴、教育陪伴，走渠道、偏高端的版本要"能拍照实物"（所以要加摄像头）。
<strong>为什么重要</strong>：这类设备的体验核心是"屏幕上那个人有多真"+"它能不能看见你/你的东西"。前者吃显示和帧率（所以 S3 撑不住），后者吃摄像头和算法。理解了体验=显示×感知，才能理解后面所有选型为什么这么定。
<strong>你能用</strong>：做任何"拟人陪伴"硬件，先把体验拆成"显示逼真度"和"环境感知"两条线，再分别去找它们对硬件的最低要求。

### 2. ESP32-S3 为什么被否：flash/RAM 是硬墙
现场原话级信息：S3"最小16兆"flash，跑数字人"最多能干到 8 到 10 帧（疑'照'为'帧'ASR误写），再多就严重卡顿（'云云云这块卡顿'）"；再叠加摄像头"基本就卡了一塌糊涂"。而且屏幕像素一旦超过某个分辨率（"2.8之后"，疑指 2.8 寸以上/某分辨率档），数字人就"不真实了"。
<strong>为什么重要</strong>：这是端侧 AI 硬件最常见的死法——不是算力不够，是<strong>内存/存储装不下高帧率视频+摄像头缓冲</strong>。帧率掉到个位数，"逼真数字人"立刻穿帮。选型时 flash/RAM 往往比标称算力更早成为瓶颈。
<strong>你能用</strong>：评估端侧芯片时，先用"你的视频帧率×分辨率+摄像头缓冲"反推内存需求，别只看 TOPS/主频。

### 3. ESP32-P4 为什么是"终极版"：翻倍且只贵十块
讲者：P4 内部 32 兆，"算力和内存都比 S 翻了一倍"，且"之前批次很贵，现在价格跌下来了，现在又差十块钱样子"。结论是"在不改方案情况下用 P4，基本是终极版"。
<strong>为什么重要</strong>：这是一个非常典型的<strong>"性价比拐点"决策</strong>——当高一档芯片降价到与低档只差极小金额、又能让你不必重做方案时，升级几乎无脑正确。它省掉的是"在 S3 上反复优化卡顿"的工程时间。
<strong>你能用</strong>：盯住你依赖的关键芯片价格曲线；一旦升级款降到"差几块且不用改架构"，立刻切，别在低端件上死磕优化。

### 4. P4 之上是 RK（RV1106）：但要"系统全改、方案全改"
讲者提到再往上就是 RK 的"RV系列 RV1106，0.5 tops，其实蛮好"，但"那样系统要改、方案全部要改"。所以本次明确止步 P4。
<strong>为什么重要</strong>：这是端侧选型里最值钱的一条<strong>架构分界意识</strong>——不是越强越好，而是"换它要不要推倒重来"。ESP32 系（MCU 生态）和 RK 系（带 NPU 的 Linux SoC）是两个世界，跨过去意味着 OS、驱动、整套方案重做。在"先跑 demo"阶段，跨界=自杀。
<strong>你能用</strong>：选型时给候选芯片标注"换它要不要改架构"。同架构内升级随便跨；跨架构升级要单独立项评估，别在赶 demo 时做。
>
>
> ⚠️ RV1106 标称算力 0.5 TOPS 为现场口径，型号/参数以 ASR 为准，选型前请查官方 datasheet 核实。

### 5. 算力放云端，板子只管采集显示（端云分工）
讲者确认垂直算法模型"不是跑在板子上"，是"领导算力的"（强烈疑为"云端算力"ASR 误写，上下文是在问"模型跑在板子上吗"）。所以板端对算力要求不高，摄像头只需"快门快、抓一张清晰照片"，两三百万像素够，省 flash。
<strong>为什么重要</strong>：这是 AI 硬件压成本的关键架构选择——<strong>重活外包给云，端侧退化成"传感器+屏幕"</strong>，于是硬件 BOM 和选型门槛都能压下来。它直接解释了"为什么 P4 就够、摄像头不用高像素"。
<strong>你能用</strong>：纠结端侧算力前先问：这步推理能不能放云端？能的话，板子选型可以降一大档，成本和功耗一起省。

### 6. "先跑 demo"的最小成本打法：现成件优先，能不自研就不自研
整通电话后半段全是这条在落地：先做 1–2 台原型机给客户演示；板子优先用<strong>现成开发板</strong>（讨论了乐心官方在卖的 86 盒成品、微雪 86 盒——但微雪"插摄像头没开孔、要自己弄口、还贵"）；外壳优先 <strong>3D 打印现成图纸</strong>（"有现成三打印图纸打成一块的最好"），不开模；<strong>不做触摸</strong>；屏幕 3.5–4 寸即可。只有都找不到现成的（带屏+带摄像头+带电池三者齐全的成品确实难凑），才考虑重新画板——而"自己画板+设计结构+3000毫安电池"会让费用明显抬高。
<strong>为什么重要</strong>：这是验证期最该有的纪律——<strong>用钱换时间的方向反了</strong>。demo 阶段每自研一个环节（画板、开模、结构设计）都是"又算设计费又算结构费"的真金白银和时间，而验证的目的只是"让客户看到、谈成需求"。
<strong>你能用</strong>：原型阶段定一条铁律——板子、外壳、图纸、电池，凡能买现成的绝不自研；把自研留到"客户确认要量产"之后。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：端侧选型按"flash/RAM 够不够 + 要不要改架构"两步走</strong>
- 核心意思：先用负载（帧率×分辨率+摄像头缓冲）反推内存够不够，再看升级要不要跨架构。
- 你可以怎么用：给候选芯片做一张表：内存够吗 / 同架构内能升吗 / 差价多少。
- 不要误读成：选算力最强的——P4 之上的 RK 更强，但要全改方案，这次反而被否。

<strong>方法 2：端云分工压低板端要求</strong>
- 核心意思：重算力放云端，板子退化成采集+显示，选型和 BOM 都能降档。
- 你可以怎么用：把每个 AI 步骤标"能否上云"，能上云的就别为它堆板端算力。
- 不要误读成：所有计算都上云——实时性/隐私强相关的（如本地抓拍）仍要留在端侧。

<strong>方法 3：demo 阶段"现成件优先"</strong>
- 核心意思：板子/外壳/图纸/电池能买现成绝不自研，先跑出能演示的原型。
- 你可以怎么用：列出 demo 所需部件，逐个先搜现成开发板/成品盒/打印图纸。
- 不要误读成：产品也用现成件凑——这只适用于验证期，量产期再谈开模/自研板。

## 📅 7 天作业

- <strong>Day 1</strong>：把你的端侧负载写清楚（视频帧率×分辨率 + 摄像头缓冲），算出最低 flash/RAM 需求。
- <strong>Day 2</strong>：列 S3 / P4 / RK 三档候选，各标"内存够不够 / 同架构能否升 / 当前差价"，复刻这次的选型表。
- <strong>Day 3</strong>：把你的 AI 流程逐步标注"能否上云"，圈出真正必须留在端侧的步骤。
- <strong>Day 4</strong>：搜一遍现成开发板/成品盒（参考乐心、微雪 86 盒），看有没有"带屏+带摄像头+带电池"接近成品的。
- <strong>Day 5</strong>：找一份能直接 3D 打印的现成外壳图纸，估算打样成本，对比"自己画板+设计结构"的费用差。
- <strong>Day 6</strong>：定义你的"最小可演示原型"——只保留让客户看懂、能谈需求的功能（这次连触摸都砍了）。
- <strong>Day 7</strong>：联系 1 个打样/开模渠道（如福建铝合金外壳卖家），问清小批量（1–2千台）的起订与价格。
