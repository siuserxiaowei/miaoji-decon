>
>
> ⚠️ 这场不是一个公司在讲一件事，是<strong>一张"2026 年 AI 硬件创业切面图"</strong>——11 个团队，11 种"怎么把硬件做成生意"的答案。如果飞书原摘要把它当成某一家产品发布会，那就没接住——它真正的价值是横向对比这 11 种打法。

## ⏱ 如果只读 10 分钟（7 条带走判断）

1. <strong>这一桌人都在回答同一个问题：硬件不赚硬件的钱</strong>。几乎每个团队都把硬件当"入口"，真正的护城河押在数据库 / 软件订阅 / 模组赋能 / token 服务上——单纯卖硬件被反复称为"会卷死"。
2. <strong>"卷它不如赋能它"成了硬件创业的显学</strong>。养老机器人柏影把核心能力（如厕清洗模组、轮椅自驾模组、感知模块）做成产品卖给传统厂家，"想做类似产品先买我的模组"——华为式打法，自己不下场卷红海。
3. <strong>找一个"政策/人口刚需"的细分，比技术领先更能活下来</strong>。柏影靠"养老是国策"拿到政府免费工厂、几千万资助、订单股东——创始人原话"在中国做养老，政府会变成世界上最好的政府"。切入点选择 > 技术堆料。
4. <strong>AI 硬件的真成本不在硬件，在 token / memory</strong>。多个团队（双一智能、Memories）指出：硬件深圳早有方案，难的是"模型记忆"和"海量视觉数据的存储/检索成本"。双一靠冷热分层 + 结构化存储把 token 消耗降 80%。
5. <strong>"安全感"是这一波硬件最大的隐形刚需</strong>。飞行机器人、视觉识别、养老监护、睡眠脑机、情绪交互——底层都在卖"安心"，而非功能本身。一句话戳穿："摄像头的本质是给用户提供安全感"。
6. <strong>陪伴 ≠ 感动，陪伴 = 持续拿到"关系成立的证据"</strong>。Sciroski（会飞 AI 宠物）把"陪伴"拆成三条证据：它存在、它因你而改变、它偏向你——这是全场最锋利的产品定义，远超"做个能聊天的玩具"。
7. <strong>隐私合规决定了硬件的存储架构，不是选型而是红线</strong>。做美国市场的团队反复强调：美国用户不让数据上云、要严格审计——所以"本地快脑 + 云端慢脑"、只存带时间戳的行为文本而非原始视频，成了被迫但正确的设计。

## 🎯 这篇真正能学什么

这场路演练的不是"认识了哪些产品"，而是<strong>横向看 11 个团队怎么把"AI + 硬件"拼成一门能活下去的生意</strong>。读完该记住四张图：①硬件靠什么后端变现（数据库 / 软件订阅 / 模组赋能 / token 服务）；②怎么选切入点（政策刚需 / 人口结构 / 隐形安全感，而非技术炫技）；③"卷 vs 赋能"两种命，以及为什么越来越多人选赋能；④"陪伴/情绪"这类软概念怎么被拆成可定价、可工程化的指标。别记谁的车更酷，记这四套思考方式。

## 📚 深挖：9 个值得单独学的知识点

### 1. "卷它不如赋能它"——华为式模组打法（全场最可迁移的法）
柏影科技（养老机器人，COO 自称闫斌/王田斌，汽车设计出身，经历通用→蔚来→小鹏）讲得最透：他不自己下场卷养老硬件红海，而是把三块硬件的核心能力——如厕清洗的"水电气冲洗烘全流程多变头"、轮椅的低速无人驾驶模组、毫米波+双目摄像头的感知模组——<strong>产品化卖给传统厂家</strong>。原话："所有想做类似产品的人，要先买我这个模组"、"卷他不如赋能他"。传统轮椅厂、传统养老厂主动来合作，他不用自建厂区。
<strong>为什么重要</strong>：当一个赛道供应链极度内卷（珠三角硬件），"做卖铲子的人"往往比"做淘金者"活得久、毛利高。这是把"被卷"转成"被买"的结构性转身。
<strong>你能用</strong>：盘点你产品里"别人也想要、但自己做很难"的那一个核心能力，把它单独抽成可售卖的模组/SDK/API。

### 2. AI 硬件的真成本是 token 和 memory，不是硬件本身
双一智能（创始人自称艾瑞/曾武，新加坡 NUS 物理博士，做室内飞行机器人）开场就点破："硬件在深圳早有好方案，难的是 token"。最大体验杀手是 <strong>memory 不够</strong>——AI 老忘记用户之前说过的，体验崩。
<strong>为什么重要</strong>：很多硬件团队把钱和精力砸在硬件 BOM 上，却忽略了真正决定体验和后端成本的是"模型怎么记住用户"。这是 2026 年 AI 硬件的隐形主战场。
<strong>你能用</strong>：做任何带 AI 的硬件/App，把"记忆架构和 token 成本"当成第一性的工程问题，别等用户量上来才发现后台被 token 拖垮。

### 3. 冷热记忆分层 + 结构化存储：把 token 消耗降 80% 的具体手法
双一给了可操作的方案，分两步：
- <strong>高低频分层（冷热记忆）</strong>：按业务方向预判用户高频问什么（如安防场景关注"出门频率/家中移动频率"），高频内容做更快检索；
- <strong>结构化 + 后台总结</strong>：检索到的内容不直接塞进 prompt，先在后台压成一句话（"用户喜欢穿红色衣服"），最终输出 token 极小；视频数据不存原始流，只存"带时间戳的行为文本"（body 姿态/attention/face 等结构化 key）。
组合下来 token 消耗降约 80%，响应也更快。对照他批评的纯 RAG："把历史对话全塞进下一轮，用户用越久成本越爆"。
<strong>你能用</strong>：别把 RAG 当默认答案；先按业务预设冷热分层，再用"后台总结成一句话"代替"原文塞回上下文"。

### 4. "陪伴 = 持续拿到关系成立的证据"——全场最锋利的产品定义
Sciroski（会飞的 AI 宠物，CEO Leo/张雨墨，康奈尔 ECE 辍学，自述养过 20+ 宠物）把"陪伴"这个被讲烂的词拆到原子级：95% 做陪伴机器人的人讲不清啥是陪伴；陪伴不是"把用户感动到哭"。陪伴=<strong>用户持续获得"关系成立的证据"</strong>，三条证据：①它存在 ②它会因你而改变 ③它偏向你（对应吵架时的三句抱怨：你不在/你不听/你不爱我）。两条通道：语言通道（其实是模型能力，跟硬件载体关系不大）和物理通道（"同一空间里有个跟你有关系的生命"）。他还点破宠物关系的真相："你第一次见你家猫狗时并没被感动"——关系来自短期多巴胺（有趣/惊喜）沉淀成长期内啡肽（安全感/被需求）。
<strong>为什么重要</strong>：这是把一个煽情概念翻译成可设计指标的范式——注意力被看见、反应的因果性、主动性、情绪共情，每一条都能落到工程上。
<strong>你能用</strong>：你产品里任何"情绪价值/陪伴感"的卖点，逼自己拆成 3–4 条可观测、可实现的"证据"，否则就是自嗨。

### 5. 选切入点：政策刚需 + 人口结构 >> 技术领先
柏影的活法：不靠技术最强，而靠"养老是国策"——拿到政府免费的南京/杭州工厂、几千万政府/银行资助、依托国家政策融钱融地、绿灯进养老院做迭代。先用"前南京首富女儿（最大养老机构）"锁定渠道订单，再做产品。原话逻辑："早期不是先做产品去卖，而是想好谁来买。"
<strong>为什么重要</strong>：在巨头林立的硬件赛道，"切一个有政策托底、有人口刚需的窄口子"比"技术更先进"更能拿到资源和订单。这是创业的资源杠杆，不是技术杠杆。
<strong>你能用</strong>：选赛道时把"有没有政策红利 / 人口刚需 / 已锁定的下单方"放在"技术是否领先"之前评估。
>
>
> ⚠️ 存疑提示：政府免费给两个工厂、几千万资助、股东下万台/2万台订单、"过亿营收已保住"——均为创始人现场路演口径，未独立核实，听做"叙事"而非"事实"。

### 6. "安全感"是这波硬件的隐形主刚需（而不是功能清单）
多个团队殊途同归：双一从第一性原理说"摄像头的本质是给用户提供安全感"，设计成"像保安一样——你出门它才巡逻，回家只给你一句'今天家里一切安全'"，而非现在摄像头矩阵给一堆"某点检测到移动"的告警让你自己点开看；Memories 主打"识别持枪/持刀的人"这种真安防；睡眠脑机、养老监护本质都在卖安心。
<strong>为什么重要</strong>：用户不想要更多信息/告警，想要"被处理掉的结论"。把功能堆成告警是负担，把功能收敛成一句安心结论才是价值。
<strong>你能用</strong>：审视你的产品是在"给用户更多需要他处理的信息"，还是在"替他处理完只给结论"——后者才卖得动。

### 7. 隐私合规直接决定硬件架构（端侧/本地存储不是选型是红线）
做美国市场的团队（双一、Memories）反复强调：美国用户不接受数据上云，即便上云也要严格审计数据流向。于是被迫也正确地走向：本地大存储（宁可售价提高一点）、只存带时间戳的行为文本而非原始视频、"本地快脑处理 + 云端慢脑分析"的快慢脑架构。
<strong>为什么重要</strong>：出海硬件的存储/算力架构往往不是技术最优解，而是合规约束下的解。先搞清目标市场的隐私法案，再定架构，否则推倒重来。
<strong>你能用</strong>：出海前先把目标国隐私合规当架构输入，端侧/数据最小化从第一版就设计进去。

### 8. "一套硬件平台 OTA 刷算法，cover 多场景"——摊薄研发成本的复用结构
速脑科技（脑机接口，创始人陈润喜）做睡眠只是第一个验证产品；底层是"神经调控=生物电监测→反馈干预的闭环"。同一套硬件，通过 OTA 刷不同算法，就能从助眠扩展到打鼾舒缓（贴下颌）、肌肉/肩颈放松——"只需做一个硬件，OTA 把算法刷进去"。
<strong>为什么重要</strong>：硬件研发昂贵，"一个平台多场景复用"能把单场景研发成本摊薄，也给融资讲了平台故事。
<strong>你能用</strong>：设计硬件时先抽出"可复用的底层能力闭环"，把不同场景做成上层算法/固件，而不是每个场景重做一套硬件。

### 9. 多模态才接得住"情绪"——单模态识别情绪是伪命题
欢创/边缘创制（CMO 周淑怡/朱淑仪）做"AI 硬件的情感引擎"：人的情感从来不是单一情绪，单模态识别接不住，所以用摄像头（面部宏观/微观表情）+ 毫米波雷达（心率/呼吸）+ 麦克风（语音语调）三模态融合，先在本地量化回归网络分析，再传大模型做跨模态融合——类比"感性脑/大脑皮层"。落地形态是给硬件厂商做 to B 的情绪解决方案（仿生猫等）。新型 AI（冯宇轩/刘宇轩）也踩到同一坑：大模型常把"撒娇"误识成"生气"，在动画呈现上是灾难。
<strong>为什么重要</strong>：凡是要做"情绪/状态感知"的产品，单模态几乎注定不准；情绪是高维信息，必须多通道交叉验证。
<strong>你能用</strong>：做情绪/状态识别，从第一版就上多模态（至少视觉+声音），并预留"误识别"的用户反馈修正回路。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：把核心能力做成模组卖给同行（卷它不如赋能它）</strong>
- 核心意思：在内卷赛道，做"卖铲子的人"比做"淘金者"活得好。
- 你可以怎么用：抽出你产品里"别人想要、自己做难"的一块，单独做成可售模组/SDK。
- 不要误读成：把整机便宜卖给别人贴牌——那还是卷；要卖的是不可替代的核心能力。

<strong>方法 2：先锁下单方，再做产品</strong>
- 核心意思：早期别先造产品再找买家，先想清楚"谁来买"并把订单/渠道锁死。
- 你可以怎么用：动手前找到一个能给你下首批订单的渠道/股东/机构。
- 不要误读成：只靠关系订单就够了——关系订单是冷启动，产品力仍要补上，否则交付即翻车。

<strong>方法 3：把"情绪/陪伴"拆成可观测的证据再去做</strong>
- 核心意思：软概念必须落成可设计、可衡量的指标（存在/因你改变/偏向你）。
- 你可以怎么用：给你的情感卖点列 3 条"用户能感知到的证据"，逐条工程化。
- 不要误读成：把"感动用户"当目标——感动是做好上述证据后的赠品，不是起点。

<strong>方法 4：记忆成本当头号工程问题（冷热分层 + 后台总结）</strong>
- 核心意思：AI 产品长期成本和体验，由"怎么记住用户"决定，不是默认无脑 RAG。
- 你可以怎么用：按业务预判高低频、做结构化存储、检索结果先压成一句话再入 prompt。
- 不要误读成：把所有历史对话都塞回上下文叫"有记忆"——那是 token 黑洞。

## 📅 7 天作业

- <strong>Day 1</strong>：把你产品的核心能力列出来，圈出"别人也想要、自己做很难"的那一个——它就是你的潜在模组/赋能产品。
- <strong>Day 2</strong>：写清你卖的到底是"功能"还是"安全感/被理解"，并把卖点改写成"替用户处理完、只给结论"的一句话。
- <strong>Day 3</strong>：如果产品带 AI，画出你的记忆架构，估算单用户长期 token 成本，设计一版冷热分层 + 后台总结方案。
- <strong>Day 4</strong>：给你产品里的"情绪/陪伴/安心"卖点，拆成 3 条用户可感知的"证据"，逐条标注怎么实现。
- <strong>Day 5</strong>：盘一个有政策红利或人口刚需托底的细分切入点，并找出"谁会先给你下单"。
- <strong>Day 6</strong>：如果要出海，查清目标国隐私合规，反推你的数据/存储架构（端侧 vs 云、存原始 vs 存行为文本）。
- <strong>Day 7</strong>：找 3 个目标用户，验证他们愿不愿为"被理解/陪伴/安心"付订阅或溢价，而不只为硬件付一次钱。
