> 场景：一通约 26 分钟的需求对接电话。甲方（张总，有车企供应商背景+内部销售渠道）想做一款"对标糯米/Normal、主打长期记忆陪伴"的车载 AI 机器人，向一家硬件方案商（多位发言人，含"胡工"）问硬件能力、AI 后端方案、报价、周期。本质是一场<strong>甲方探底、方案商报实情</strong>的合作前对齐。
>
>
> ⚠️ 此稿无飞书原 AI 摘要可比对；但 ASR 把品牌名识别得很乱（"糯米/农民"=糯米，"小度熊/百度熊"=小度，"小智""华南理工复刻"等），人名口径都要打折看。

## ⏱ 如果只读 10 分钟（6 条带走判断）

1. <strong>这类"AI 陪伴硬件"的核心卖点已收敛到一个功能：长期记忆</strong>。甲方反复强调"最主要是长期记忆"，转动、联网、对话都排在它后面。谁把"记得住用户说过什么、并据此回应"调好，谁就拿到了这一代陪伴硬件的命门。
2. <strong>AI 后端不是自研大模型，而是"包一层服务端去调大厂 API"</strong>。方案商坦白：他们没训练模型，就是写了套服务端适配百度云的 API。这是当下中小硬件团队做 AI 的真实姿势——拼的是集成和成本，不是模型。
3. <strong>大厂语音云的报价模式是"按设备一次性收费、永久使用"</strong>，不是按对话量。一个设备几十块、聊到天荒地老不再加钱。这直接决定了硬件能不能量产——按时长计费会被一小时几十块的成本拖死。
4. <strong>小智生态要授权得交押金/授权费，门槛十几万到二十几万</strong>，且每个商务报价都不一样。想用现成生态先去谈商务，别先开发。
5. <strong>硬件没有"一个月出样机"这种事</strong>。方案商划线：最小可用样机 3 个月，样机到量产再半年起步，"最后 20% 花的时间最长"。甲方想压缩到 1 个月被直接否掉。
6. <strong>这一代陪伴硬件的竞争正从"产品"转向"宣传/品牌"</strong>。方案商点名"虾线科技（花衍）产品没上线就攒了十几个粉丝群"，反衬"包个壳就卖、退货率极高"的小作坊。甲方的底气是车企采购渠道+内部销售，不靠众筹。

## 🎯 这篇真正能学什么

这不是技术评审，是一张<strong>"想做 AI 硬件的人，第一通对接电话该问什么、会被怎么报实情"的清单</strong>。值得学三件事：①AI 陪伴硬件的真实技术栈长什么样（服务端调大厂 API + 大厂按设备买断计费 + 生态授权押金）；②硬件项目的真实周期与"快不了"的结构性原因；③甲方靠什么活下来（渠道/记忆功能/宣传），以及方案商怎么自证靠谱。读完别记参数，记这三张"做硬件的现实图"。

## 📚 深挖：7 个值得单独学的知识点

### 1. 这一代陪伴硬件的命门只有一个：长期记忆
甲方在整通电话里至少四次把需求收敛到"最主要是长期记忆"——能转动、能联网、能对话都排在它后面。逻辑是："这个陪伴，就是我能记忆中之前说过什么，然后会有一个响应。"
<strong>为什么重要</strong>：这是把"语音玩具"和"陪伴机器人"分开的那条线。没有长期记忆，每次对话都是冷启动；有了，用户才会产生"它认识我"的依恋，复购和留存才立得住。而且方案商点破——这功能"你要调好，后面其他都能用"，它是杠杆点。
<strong>你能用</strong>：做陪伴类产品，先把"记得住+据此回应"这一个功能调到人性化，再谈别的花活。

### 2. 中小团队的 AI 不是自研模型，是"服务端包一层调大厂 API"
方案商被追问时反复澄清："也没有训练到 A 整，就是直接做了一个服务端调他们的 API 而已""我们的 API 适配百度云这边调的 API"。所谓"做了一个 agent"，本质是基于百度云服务重写了一套服务端做适配。
<strong>为什么重要</strong>：这戳破了很多"AI 硬件公司"的滤镜——绝大多数不碰模型训练，拼的是集成效率、成本控制和谁家大厂 API 更便宜更稳。理解这层，你才能正确评估一家硬件团队的真实技术含量。
<strong>你能用</strong>：评估/自建 AI 硬件方案时，先分清"调 API 适配"和"自研模型"，前者的护城河在成本和工程，不在算法。

### 3. 大厂语音云的真实计费：按设备买断，不按对话量
这是全场最实的一条成本知识。方案商解释为什么不能用火山/豆包按时长那套——"你通话个一个多小时就几十块了"，量产成本爆炸。而百度云/小智是"按设备收费，一个设备几十块，以后一直聊都不再收费"，永久。百度云约 20 元出头还含视觉功能，响应 1 秒多。
<strong>为什么重要</strong>：计费模式直接决定一款 AI 硬件能不能量产。按 token/时长计费的方案在 C 端高频陪伴场景里会被成本拖垮；按设备买断才能把边际成本压到可控。这是选型时第一个该问的问题。
<strong>你能用</strong>：选语音/对话云，第一句先问"是按设备买断还是按对话量"，后者在陪伴场景基本判死刑。

### 4. 用现成生态（小智）的门槛是押金/授权费，先谈商务再开发
甲方提到想用小智 AI 授权，方案商立刻提醒"要押金，押金 20 万""或者授权费"，而且"每个商户报价都不一样"——他聊过两个，一个十几万、一个二十几万。
<strong>为什么重要</strong>：很多人以为"用开源/现成生态=省钱省事"，但生态授权的商务门槛可能比自己包一层还高，且不透明。华南理工复刻的开源版被方案商直接判"完全不能用"，免费的坑更大。
<strong>你能用</strong>：想接任何现成 AI 生态，第一步是找它的商务谈清押金/授权/分成，别先投开发，否则可能白做。

### 5. 硬件"快不了"是结构性的，样机≠可量产
甲方想"一个月出样机"被反复否掉。方案商给的真实节奏：最小可用样机 3 个月，"样机到量产之前最少半年"。原因说得很透：样机做到 80% 不难，<strong>剩下 20% 花的时间最长</strong>——要各种测试、派工程师去工厂对接培训驻场。"有的客户以为样品拿出来就能量产，那不行的。"
<strong>为什么重要</strong>：这是硬件创业最容易被低估的现实。软件可以热更新迭代，硬件的打版、测试、产线导入是物理时间，抄方案也省不掉打版时间。误判周期会直接拖垮资金节奏。
<strong>你能用</strong>：做硬件排期，把"样机→量产"单独留半年以上，并把最后 20% 当作最难的部分预留资源。

### 6. "拿样机撬资金"的资金节奏设计
甲方说得很直白：目标是"1 个月～1 个半月把押金/样机拿到→给领导展示→才能有大资金注入→然后搞量产"。样机此刻不需要精美，"哪怕 3D 打印壳子、临时壳子能转动"，只要把长期记忆这个核心想法演示出来就达标。
<strong>为什么重要</strong>：这是一种典型的"用最小可演示成果解锁下一笔钱"的节奏。资金不是一次到位，而是被"可见的进展"一段段释放。理解这点，就知道为什么甲方愿意牺牲精美度换速度——他要的是过领导那一关。
<strong>你能用</strong>：缺资金时，先定义"能解锁下一笔钱的最小可演示成果"，集中火力做它，别追求一步到位的完美品。

### 7. 竞争正从产品转向宣传/渠道
方案商主动点出行业现状：一类是汕头澄海的老工程师"包个壳就卖"，送免费 token，数据好看但<strong>退货率极高、外壳很垃圾</strong>；另一类是"虾线科技（按口径应为花衍科技）产品还没上线，就拉了十几个粉丝群"，靠新春宣传起势。甲方的底气则是第三种——车企采购渠道+内部销售运营，"至少能做车机捆绑销售，不用去众筹拉粉丝团"。
<strong>为什么重要</strong>：当硬件功能和方案趋同，差异化战场前移到了宣传声量和渠道保底。三种活法（铺量包壳/宣传攒粉/渠道捆绑）对应三种风险结构，选哪条决定你怎么分配资源。
<strong>你能用</strong>：做硬件先想清楚自己靠哪条活——产品力、宣传声量、还是渠道保底，别三样都想要却都不强。
>
>
> ⚠️ "虾线科技"按 ASR 极可能是"花衍科技"，做车载机器人、宣传攒粉群的描述与之吻合，但请自行核实。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：核心功能优先，外观/精美度全部后置</strong>
- 核心意思：第一版样机只验证最核心的功能（这里=长期记忆+能转+能对话），壳子临时凑。
- 你可以怎么用：把产品拆成"撬动下一步所必需的最小功能集"，先做这个。
- 不要误读成：可以一直糊弄下去——方案商也提醒"想卖得好就不能做太差"，精细化和快速上线不可兼得，临时壳只用于演示阶段。

<strong>方法 2：AI 后端按"计费模式"而非"模型名气"选型</strong>
- 核心意思：选语音/对话云先看是不是按设备买断、响应延迟多少、含不含视觉，而不是看谁家模型听起来牛。
- 你可以怎么用：列一张表对比百度云/声网/火山的"买断价+延迟+功能"。
- 不要误读成：越大牌越好——火山官网 6.5 折仍比百度贵，名气不等于适合量产。

<strong>方法 3：用"最小可演示样机"解锁资金，而非等全做完</strong>
- 核心意思：先做一个能演示核心想法的样机去过决策人那关，换取下一笔资金。
- 你可以怎么用：定义你的"演示即解锁"里程碑，集中做它。
- 不要误读成：样机做出来就等于能量产/能卖——这是两回事，后面还有半年量产路。
