## 📚 深挖：5 个值得单独学的知识点

### 1. 在线唤醒词 vs 离线唤醒词：一颗芯片的取舍
方案商原话："离线的话，如果你要定制，可能要加个三四块钱，因为那个离线芯片就要三四块钱；你要在线的话，就可以省去这个钱，就不需要芯片了。"
- <strong>是什么</strong>：唤醒词（喊"小智小智"那种）有两种实现。离线靠板上一颗专门的离线语音识别芯片，断网也能唤醒；在线把唤醒交给联网服务，省掉这颗芯片。
- <strong>为什么重要</strong>：这不只是省 3-4 元的问题。<strong>离线芯片往往把唤醒词烧死、改不了</strong>（见下一条），这正是客户最大的痛点来源。在线方案则天然可改。
- <strong>你能用</strong>：如果你的产品需要"自定义/可改唤醒词"或想做品牌差异化叫法，优先在线方案；如果场景要断网可用、对唤醒词无所谓，离线更省。

### 2. 离线语音芯片 = 唤醒词被"焊死"的真凶
客户反复说"现在不能改唤醒词比较头疼""今天下午我发现不能改这个唤醒词，我就有点懵"。方案商看了板子背面"铁皮框起来"的模组后判断："你这是离线语音的芯片，难怪不能修改。"
- <strong>是什么</strong>：客户那块现成板（自称工厂设计的"小智模式"开发板）用的是离线语音芯片，唤醒词在出厂时就固化进芯片，二次开发改不动。
- <strong>为什么重要</strong>：这是<strong>外行选型最隐蔽的坑</strong>——板子便宜、能跑，但灵活度被硬件层锁死，到了要做品牌/改交互时才发现改不了，已经设计完外壳、采购完，很被动。
- <strong>你能用</strong>：买现成 AI 语音开发板，第一件事就问"唤醒词是不是烧在离线芯片里、能不能改"；别等外壳都做好了才发现锁死。客户已经在考虑的退路是：干脆不改唤醒词，对外叫"小智二次开发产品"。

### 3. "0.5 秒延迟"的真相：固定问答才有，联网过大模型做不到
客户拿小智开发板"基本都在 0.5 秒"焦虑，问方案商能不能做到。在场三人轮流点破：
- 方案商："一两秒目前肯定，如果不是特殊处理，应该做不到 0.5 秒。"
- 发言人3："你问的问题只要联网搜索、要过大模型的，都没有能控制在 0.5 秒。现在哪个大模型接了之后能达到 0.5 秒？没可能。"
- 方案商："他能立马拿出来，是因为'一加一等于几'这种他自己训练过的；你说的不是平常用的，他也需要那么久。"
- <strong>是什么</strong>：低延迟只在"本地预置/训练过的固定问答"成立；任何要联网检索或走云端大模型的，链路决定了就是 1-3 秒。
- <strong>为什么重要</strong>：这是<strong>对标焦虑的解药</strong>。外行容易拿竞品某个极端场景的数字（0.5秒）当通用标准，逼自己的供应商达标，其实是拿苹果比橘子。
- <strong>你能用</strong>：评估语音产品延迟，先区分"本地固定问答"和"联网大模型问答"两类，分别看；别用前者的数字要求后者。

### 4. 你的"游戏"恰好是固定题库——根本不需要低延迟
方案商连着两次点破客户的真问题："你这个游戏题目是固定的、答案也是固定的（一加一等于几，答固定），那就不需要那么快，不需要联网想。"客户的"专注力游戏"正是 ABCD 四选一、题目和答案都预置（有的题还要配插图）。
- <strong>是什么</strong>：客户产品的核心交互（答题游戏）是确定性问答，可以本地预置，天然能快。
- <strong>为什么重要</strong>：客户把"卡顿/延迟"当成头号风险在到处问，但他真正要做的功能根本不吃延迟——<strong>风险点找错了，焦虑用错了地方</strong>。他真正的卡顿问题方案商判断是"服务器软件"的事，跟板子关系不大。
- <strong>你能用</strong>：在为某个指标焦虑前，先确认你的核心场景到底吃不吃这个指标。固定题库做本地化，延迟不是约束。

### 5. AI 玩具板的 BOM 心算 + 方案商两种报价结构
<strong>BOM（物料）拆解</strong>（均为口头报价，存疑，且随量浮动）：不带屏带喇叭带电池约 33-35 元（1200mAh 电池 + 喇叭）；屏单买约十几到 23 元（1.8寸 23 元，1.5寸便宜约 10 元，不带触摸）；客户现有整板（屏+喇叭+电池）约 49-50 元。客户给方案商定的成本红线：<strong>不带电池、带屏，控制在 50 元以内</strong>；方案商估"带屏板子三四十元左右，量小（100个）拿不到便宜区间"。
外设清单（方案商复述）：type-c + 喇叭 + 麦克风 + 离线语音 + C3主控 + 屏 + 长按开关（短按易误触，一般长按 3 秒）。
<strong>两种报价结构</strong>（这是最该记的）：
1. <strong>交研发费</strong>：给你单独设计板子收研发费，<strong>无起订量</strong>，灵活。
2. <strong>不收研发费</strong>：但要 <strong>1K 起订 + 签合同 + 打 1000 预付款</strong>（方案商提前垫人力，所以要锁量）。
客户首批只想做 100 个（几个博主分着卖测市场），还问"研发费+软件开发费总价一万行不行""服务器后续 10 元/台"。
- <strong>为什么重要</strong>：这是硬件定制的<strong>通用报价逻辑</strong>——研发是一次性人力成本，要么你付研发费、要么用起订量摊。量小又不想付研发费，是拿不到的。
- <strong>你能用</strong>：找方案商定制前，先想清楚自己是"小批量试错"（认研发费、保灵活）还是"已确定放量"（吃起订量、省研发费）；并直接把成本红线报给对方让其倒推，比逐项砍价高效。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：低成本验证供应商——先要视频/照片，别急着买样品</strong>
- 核心意思：客户想打样，方案商说"我给你拍个视频/你也不用买"，还指导客户拍板子背面的铁框模组来判断芯片类型。
- 你可以怎么用：评估硬件供应商，先要一段真实使用视频看延迟/卡顿体感，再决定是否花钱打样。
- 不要误读成：视频能替代打样——视频只能初筛体感，真要量产前还得拿到实物测。

<strong>方法 2：把成本红线直接抛给方案商，让他倒推</strong>
- 核心意思：客户最后用的是"不要电池、带屏、50 元以内能不能做"这种给红线的问法，比逐项问价高效。
- 你可以怎么用：谈定制时直接报"我整机/板子成本要控制在 X 元，含哪些外设"，让对方回可行/不可行。
- 不要误读成：红线越低越好——量小时（100个）压不到极致成本，给不切实际的红线只会被拒。

<strong>方法 3：选型先问"唤醒词能不能改"</strong>
- 核心意思：唤醒词改不改得了，由是不是离线芯片/是否烧死决定，是个硬件层约束。
- 你可以怎么用：采购现成 AI 语音板，第一句就问唤醒词实现方式和可改性。
- 不要误读成：所有板子都能改唤醒词——离线芯片方案大概率改不了。
