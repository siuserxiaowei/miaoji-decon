## ⏱ 如果只读 10 分钟（7 条带走判断）

1. <strong>硬件能不能返修，是在外观阶段就被结构决定的，不是事后补的</strong>。甲方一开始还在纠结"外面要不要可拆卸"，结构方一句话点破：里面做成模块化（主板引出接口、各模块用排线连），坏了换模块就行；但<strong>外壳必须能打开</strong>，否则板子塞不进去、坏了也修不了。
2. <strong>模块化是硬件返修的标准答案</strong>：一个盒子装主板，主板引出接口，屏幕/小夜灯/雷达全用排线接出去，哪个模块坏换哪个。这是"现在都这样设计"的行业默认，不是创新。
3. <strong>"几块钱的东西没必要修"</strong>——发言人3 的这句话其实是个成本决策原则：返修设计是为了换贵模块（主板、屏幕），不是为了修便宜耗材；为省几块钱搭进去的拆装成本可能更高。
4. <strong>灯要透光 vs 外壳要够硬，是一对真实矛盾</strong>。甲方想用食品级硅胶直接透光，结构方反复说"硅胶不够硬、透出来不好看"。最后收敛到<strong>双层方案</strong>：里层树脂/3D 打印硬壳裹元器件，外层套一层可调色的透明软硅胶，整体像光纤/发光体。
5. <strong>"分两层"听着完美，但硅胶那层甲方得自己找人调</strong>。结构方明确说软硅胶套这种场景感应材料"我们这边不好做"，他们只能做里层 3D 打印树脂硬壳 + 塞硬件；外层透明可调色硅胶要甲方自己找材料商。<strong>别把"方案可行"误读成"对方全包"。</strong>
6. <strong>底部开模拆装会被"倒钩/倒角"卡死</strong>——这是全场最值钱的工艺细节。甲方想从底部上螺丝拆，结构方画图警告：底面如果有回勾/倒角，灌胶进去后<strong>脱模时拉不出来</strong>。解法要么把分割线上移到没倒角处，要么干脆把底做成直上直下不带圆角。
7. <strong>流程上先交外观、再谈结构</strong>：结构方定调——甲方只要先给一个大概外形（STP 文件），他们的结构工程师再据此定哪里开模、哪里加减；"你想得出来，生产上不一定做得出来，要不停跟（厂）磨"。

## 🎯 这篇真正能学什么

这不是一场商业模式会，是一场<strong>"把一个 AI 硬件的外观图，落到能量产、能返修的结构"的实战推演</strong>。值得学的是三种眼力：①<strong>返修性是设计出来的</strong>——模块化 + 可开壳必须在外观阶段就想清楚；②<strong>工艺约束会反杀创意</strong>——透光/硬度、脱模/倒角、上色/材料，每一个"用户体验上想要的"背后都有一个"工厂做不做得出来"；③<strong>甲方与供应链的责任边界</strong>——哪段对方包、哪段你自己找人，要在对接时就问死。读完别记"他们聊了个玩偶"，记住这三张"从外观到量产的避坑图"。

## 📚 深挖：6 个值得单独学的知识点

### 1. 模块化设计：硬件返修的物理底座
<strong>是什么</strong>：发言人2 反复讲的核心架构——"一个盒子里装主板，主板引出各种接口，屏幕、小夜灯、雷达这些模块用排线（电子线）连出去"。哪个模块坏了，整个模块换掉，不修单个元件。发言人3 补一句"现在都是这样子的设计"，说明这是行业默认而非新点子。
<strong>为什么重要</strong>：它把"返修"从"拆开整机焊电路"降维成"拔插一个模块"，既降售后成本，也降对用户/客服的技术门槛。这是所有可维护硬件的通用骨架，能迁移到几乎任何带主板+外设的电子产品。
<strong>你能用</strong>：设计硬件时，先画"主板—接口—排线—模块"这张拓扑图，把"坏了换哪块"想清楚，再谈外壳。

### 2. "几块钱的东西没必要修"——返修的成本边界
<strong>是什么</strong>：当甲方纠结里面的小件能不能也做成可拆，发言人3 直接说"那没必要那么几块几块钱的东西，没必要修那个东西"，发言人2 补"你还要拆那个（外层），那成本就更高了"。
<strong>为什么重要</strong>：返修设计不是越细越好。为一个几块钱的件做可拆结构，搭进去的模具/装配/人工成本会远超那个件本身。返修要卡在"贵且易坏的模块"（主板、屏幕）上，对便宜耗材直接放弃维修、整体替换。
<strong>你能用</strong>：列你产品的 BOM，按"单价×故障率"排序，只为高价高故障的件做可换设计，其余别浪费结构预算。

### 3. 透光 vs 硬度的矛盾，与"双层壳"解法
<strong>是什么</strong>：甲方想用食品级硅胶让小夜灯的光直接透出来，发言人2 连续反对："硅胶不够硬""透出来不好看"。来回拉扯后收敛到双层方案——"外面做一层硅胶套，里面再做实的（树脂硬壳）""里面可以做透明的树脂硬一点，包住元器件，外面再套一层很软的透明硅胶"，效果"跟光纤一样，整个会发光"。同时点出新风险：纯软硅胶用户一捏，可能把里面的电路捏坏。
<strong>为什么重要</strong>：这是工业设计里典型的"两个用户诉求互斥（要软糯手感 + 要透光 + 要保护电路），单一材料满足不了，用分层结构拆解"。这种"用结构分层化解材料矛盾"的思路可迁移到很多软硬结合产品。
<strong>你能用</strong>：当一个零件被要求同时满足互斥属性时，先问"能不能拆成两层/两件各管一个属性"，而不是逼一种材料全扛。

### 4. 脱模与倒钩/倒角——最容易被外观党忽略的量产杀手
<strong>是什么</strong>：甲方想从底部开口、上螺丝拆装。发言人2 反复警告：底面如果是带圆弧/回勾的造型，灌胶（硅胶/树脂）进模后，<strong>脱模时那个倒角/弯角会把件卡住、拉不出来</strong>（"你那个膜可不好拖出来""这两个脚不就很容易卡住了吗"）。解法两条：要么把拆装的分割线"往上移到没有倒钩的位置"，要么"底面不要做成圆的，直接做一个直上直下的"。
<strong>为什么重要</strong>：这是只有干过结构/模具的人才会提前喊停的细节。外观设计师/产品经理画图时只想"好看、好握"，但<strong>倒钩、倒扣、负角度在注塑/灌胶里就是脱模灾难</strong>，轻则做不出、重则模具报废。这一条本身就值这场会。
<strong>你能用</strong>：任何要开模的外观，定型前让结构/模具方过一遍"有没有脱模负角度"；拆装分割线优先放在直壁、无倒钩处。

### 5. 责任边界：哪段供应链包、哪段甲方自己找人
<strong>是什么</strong>：双层方案聊嗨时，发言人2 踩了刹车——软硅胶外套这种"场景感应"材料"这个可能我们这边不好做"，他们只能做"里那层硬的，3D 打印那个树脂外壳，把硬件塞进去"；外层透明可调色硅胶"要你自己去调配、自己去找那种材料"。甲方追问"这个还是得找人对吧"，对方答"对"。
<strong>为什么重要</strong>：一个看起来"方案可行"的结论，落到执行时往往被切成"我做这段、你自己搞那段"。如果对接时不把边界问死，甲方很容易误以为对方全包，到交付才发现一大块要自己另找供应商，进度和成本全乱。
<strong>你能用</strong>：每次供应链对接，对每个工序明确问"这段是你做还是我找人"，把责任边界写进对接结论，别停在"方案可行"。

### 6. 先交外观、再定结构——硬件协作的正确推进顺序
<strong>是什么</strong>：会议尾段发言人2 定调流程："你把外观先提供，我们这边结构工程师会根据你这个外观做修改，到底哪里开模、哪里什么……你先提供一个大概的外形（STP），后面再去协商哪里加、哪里减。""这个东西因为有很多工程上的问题，你可能想出来了，但不一定在生产上能实现，要不停地跟（厂）磨。"甲方接受，承诺"明天大概能给（外观），后面再协商怎么拆件"。
<strong>为什么重要</strong>：很多甲方想一次把外观+结构+拆装全定死再交付，结果反复返工。正确节奏是<strong>外观先粗交→结构方介入做工程性修改→再迭代细节</strong>，因为"想得出来"和"做得出来"之间隔着大量工程约束，必须靠双方来回磨。
<strong>你能用</strong>：硬件项目排期按"外观草案→结构评审→开模细节"三段走，第一段别追求完美，先让结构方尽早介入挑刺。

## 🛠 可直接复用的方法（含"不要误读成"）

<strong>方法 1：模块化 + 可开壳，作为返修默认架构</strong>
- 核心意思：主板引接口、排线接模块、外壳可打开，坏了换模块不修整机。
- 你可以怎么用：先画主板—模块拓扑，再设计一个可拆的外壳开口（底部上螺丝或对半分）。
- 不要误读成：所有件都要可拆——便宜耗材不值得做可换设计（见知识点 2）。

<strong>方法 2：用"结构分层"化解材料互斥诉求</strong>
- 核心意思：要软手感+要透光+要护电路，单材料扛不住，就里层硬壳外层软套分工。
- 你可以怎么用：遇到一个零件被塞多个互斥要求时，先想能不能拆成两层各管一项。
- 不要误读成：分两层就万事大吉——外层那种特殊材料可能供应链不包，得自己找人调（见知识点 5）。

<strong>方法 3：定拆装分割线前，先排脱模负角度</strong>
- 核心意思：底部/分割处若有倒钩倒角，脱模会卡住拉不出，必须避开。
- 你可以怎么用：把分割线放在直壁无倒钩处，或把底改成直上直下。
- 不要误读成：分割线随便放在"用户体验好"的位置——好握的弧面往往正是脱模灾难区。

## 📅 7 天作业

- <strong>Day 1</strong>：画出你这款（或任一对标）硬件的"主板—接口—排线—模块"拓扑图，标出哪些模块会坏、坏了怎么换。
- <strong>Day 2</strong>：对 BOM 按"单价×故障率"排序，圈出"值得做可换设计"的件，其余标"整体替换不返修"。
- <strong>Day 3</strong>：把外壳的拆装方案画两版（底部上螺丝 vs 对半分），各标一条优点和一条脱模风险。
- <strong>Day 4</strong>：找出你外观里所有"倒钩/倒角/负角度"位置，逐个问"脱模拉得出来吗"，标红改不掉的。
- <strong>Day 5</strong>：如果有透光/手感/保护互斥需求，画一版"双层壳"分层方案，注明每层材料和谁来做。
- <strong>Day 6</strong>：列一张"工序责任边界表"：每道工序写清"供应链包 / 我自己找人"，把模糊处标待确认。
- <strong>Day 7</strong>：准备一个"大概外形"的 STP（或手绘+尺寸），约结构方做一次早期评审，专门请他们挑工程实现的刺。
