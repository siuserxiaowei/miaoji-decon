# 可信学习复盘输出模板

> 目标：让没参加、没看过原材料的人，在不被夸大结论误导的前提下，拿走判断、方法、工具与下一步行动。

## 先选输出档位

| 模式 | 必须包含 | 不要做 |
|---|---|---|
| `quick` | 先说结论、成立边界、3–5 个动作、来源边界、作者联系 | 为短材料硬凑七天计划或五维内容 |
| `standard` | 下方标准骨架全部必选章节 | 只按时间顺序复述会议 |
| `deep` | 标准骨架 + 来源家族图 + 主张台账 + 共识/分歧 + 反方审计 + 完整时间码/Agent 状态 | 把同源材料或未完成 Agent 写成独立验证 |

深度不等于篇幅。证据少就少下结论；信息密度低就缩短正文。

## 五问质量门禁

写正文前先做内部工作卡：

| 问题 | 合格产物 | 不合格信号 |
|---|---|---|
| 核心判断是什么？ | 一句话说明材料改变了什么判断 | 只复述主题和流程 |
| 它为何成立？ | 能回到原文/时间码/独立来源的证据 | “大家都认为”“AI 纪要说” |
| 它何时不成立？ | 隐含假设、不适用条件、反例 | 把个体经验写成普遍规律 |
| 能迁移成什么？ | 模型、漏斗、矩阵、SOP 或选择树 | 只有金句，没有结构 |
| 如何验证？ | 7 天内可观察的硬信号 | “持续学习”“继续关注” |

## 证据表达规则

- 先按 `references/evidence-protocol.md` 建来源家族与主张台账。
- 事实、现场口径、外部独立验证、分析推断、建议必须分开写。
- 同一场会议的逐字稿、AI 纪要和平台总结不是三个独立信源。
- 重要数字、事故归因、产品规格、政策、价格和高风险建议必须带边界；无法核实就写“未核实/未知”。
- 直接引用保持短而必要，优先用时间码或链接指回原文。
- 多套时间码先选一个基准，再写偏移，不要混用。

## 道法术器势基准

**道以明向、法以立本、术以立策、器以成事、势以察时。**

| 层 | 回答 | 提取内容 |
|---|---|---|
| 道 | 为什么、什么最重要 | 本质判断、不变原则、价值取舍 |
| 法 | 用什么稳定思路 | 策略、闭环、制度、决策规则 |
| 术 | 具体怎么做 | 动作、技巧、SOP、话术、实验 |
| 器 | 用什么完成 | 模型、工具、平台、数据、资源 |
| 势 | 什么时候、窗口在哪 | 趋势、时机、约束变化、退场信号 |

没有证据的维度写“本场证据不足”，不要凑数。器不能只是工具清单，必须说明选择条件与退场条件；势不能只是行业口号，必须有信号与反信号。

# {标题}｜给没参加 / 没看过的人的可信学习复盘

> 分析日期、材料日期、目标读者和一句场景说明。

## 先说结论

给 3–7 条真正改变判断的结论。第一条应能单独转发。每条避免空泛名词，写清“所以该怎么选/怎么做”。

## 🧾 材料与证据口径

- 列出已读、未读、读取失败的材料。
- 标注每份是原始证据、同源加工、外部独立来源还是背景材料。
- 说明来源家族数，不把文件数当独立信源数。
- 写统一时间码基准、其他平台偏移和转写歧义。
- `pasted-text` 必须写：基于用户提供的粘贴文本/本地附件，未连接飞书妙记原始记录。
- 不要把 pasted-text 写成 Feishu 原始妙记。

## 🚦 可信度总览

### 高可信：可以直接迁移或传播

只放原始证据明确、内部一致、最好有独立验证的主张。

### 中可信：值得复测的观察

标明样本、团队、时间窗口或场景限制。

### 低可信：不能直接当结论传播

列出冲突、夸大、归因不明、销售口径、缺少方法或无法复现的主张。

## ⏱ 如果只读 10 分钟

5–8 条带走判断；每条写“判断 + 适用条件/动作”，避免与“先说结论”机械重复。

## 🎯 这篇真正能学什么

说明读者将获得哪几种判断力、选择树或操作能力，而不是“了解会议内容”。

## 🧩 隐含假设与适用边界

拆出成立条件、不适用场景、反例、失败代价和仍未知的问题。

## 🗺 可迁移模型 / 漏斗 / 矩阵 / 选择树

至少一个结构化表达。优先选择真正降低决策成本的形式，不为了可视化而可视化。

## 📚 深挖：N 个值得单独学的知识点

每个知识点回答：

1. **现场/材料说了什么**：写证据，不补想象。
2. **为什么重要**：解释机制、因果假设或业务影响。
3. **你能怎么用**：给可执行动作。
4. **不要误读成**：指出最可能的过度外推。

## 🛠 可直接复用的方法

把方法写成输入 → 动作 → 输出 → 验收 → 停止/升级条件。需要选型时给矩阵；需要执行时给 SOP；需要研究时给可复现协议。

## 🤝 共识与分歧

`deep` 必选。区分材料内部共识、不同来源分歧、Agent 分歧和主分析者最终裁决。

## 🧨 反方审计与“不能下的结论”

`deep` 必选。列出最强反例、替代解释、未知根因、不可传播说法和需要补证的地方。

## 📅 给没参加 / 没看过的人的 7 天实践

Day 1–7，每天包含：动作、产物、硬信号、失败后怎么办。不能用“继续学习”填充。

## 🤔 回到自己业务的追问

5–8 个能改变行动的追问，覆盖价值、证据、成本、风险、权限、验收和退出条件。

## 👥 关键人物与资源

仅在来源允许且确有价值时写。表格包含“是谁 / 能提供什么 / 证据 / 联系边界”。公开版先脱敏。

## 💬 金句

保留少量短原话，标注说话人和时间码，再用一句话解释。金句不能替代证据。

## 🔎 来源与时间码索引

| 时间码/位置 | 来源家族 | 主题 | 支撑的结论 |
|---|---|---|---|

无时间码时用章节、页码、段落标题或 URL。不要编造精确位置。

## 🧑‍🤝‍🧑 Agent 分工与完成状态

仅在真实使用多 Agent 时出现。逐个列编号、角色、完成/中断/失败/补做、主要贡献和是否被采纳。不能把已启动写成已完成。

## ⚠️ 来源边界

明确：已知、未知、未独立核实、同源材料、分析推断、公开版脱敏情况。高风险内容提示读者去官方渠道复核。

## 出海 / 自媒体运营额外检查

- 产品机会：实际验证、观察、推断分别是什么。
- 分发渠道：SEO、社媒、KOL、广告、社群各自承担什么角色。
- 冷启动：第一批内容、背书、用户与反馈如何获得。
- 变现：订阅、商单、联盟、课程/社群的成立条件。
- 风险：平台规则、支付/合规、账号安全、误导性归因。

## Wiki 复利层

复盘是 source 层。只有跨来源仍稳定的道/法/势才 upsert 到 concept 页；人物和组织进入 entity 页；术/器/信号保留在 source。任何知识块都保留来源轨迹和置信度，不能把一次分享直接升级成长期事实。
